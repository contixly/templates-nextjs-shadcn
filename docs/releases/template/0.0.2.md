# v0.0.2

Internationalization release for the Next.js application template.

This release captures the `main` branch state published at commit `2b91afb` and builds on `v0.0.1` by adding a reusable localization foundation across the main template flows.

## Highlights

- Added `next-intl` to the template stack
- Introduced English and Russian message catalogs for the main feature areas
- Localized page titles, descriptions, and Open Graph metadata
- Refactored core UI copy to resolve through translation namespaces instead of hard-coded strings
- Expanded test coverage for i18n helpers, metadata generation, forms, runtime components, and environment handling

## Included in this release

### Internationalization foundation

- `next-intl` integrated through `next.config.ts` and request configuration
- Locale configuration for `en` and `ru`
- Typed message declarations for shared and feature-scoped translation files
- Translation loaders and shared helpers for server and client consumers

### Localized application flows

- Public home page localized
- Auth pages and account settings flows localized
- Workspace management screens localized
- Dashboard-related UI copy localized
- Shared navigation, breadcrumbs, dialogs, forms, and error states updated to consume translations

### Localized metadata and page definitions

- Page translation namespaces added to route/page definitions
- Metadata helpers updated to resolve localized titles and descriptions
- Open Graph and Twitter metadata aligned with localized page copy
- App-level documentation and UI copy adjusted to reflect the translated template structure

### Runtime and environment refinements

- Yandex Metrika counter env renamed to `NEXT_PUBLIC_YM_COUNTER_ID` for client-safe access
- Global error fallback hardened
- Supporting environment examples updated for locale and analytics configuration

## Quality and verification

- Added tests for i18n request setup and page translation helpers
- Added tests for localized metadata exports and translated runtime components
- Added tests for localized forms and schema-backed validation copy
- Added regression coverage for environment access and global error behavior

## Upgrade notes

Compared with `v0.0.1`, template users should note:

- `next-intl` is now part of the default stack
- Translation files live under `src/messages/`
- The default locale can be configured with `PUBLIC_DEFAULT_LOCALE`
- Yandex Metrika now expects `NEXT_PUBLIC_YM_COUNTER_ID` instead of the previous server-only variable name
