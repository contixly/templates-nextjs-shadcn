# v0.0.8

Local automation and Playwright E2E release for the Next.js application template.

This release builds on `v0.0.7` by adding a local-only Better Auth automation login flow for browser-driven
development agents, introducing Playwright end-to-end test infrastructure, and hardening organization invitation
policy checks at the Better Auth plugin boundary.

## Highlights

- Added opt-in local automation authentication for Playwright, browser-use, and LLM-driven development agents
- Added `/api/local-auth/scenario` to create signed-in automation users and clean them up after local scenarios
- Added a gated login-page automation panel for one-click local test-user creation
- Enabled Better Auth email/password only outside production when `LOCAL_AUTOMATION_AUTH_ENABLED=true`
- Added generated automation credentials in the `local-agent+...@local-agent.test` namespace
- Added Playwright configuration, scripts, browser install command, report commands, and Chromium smoke coverage
- Added Playwright MCP server configuration for local browser automation workflows
- Added E2E support fixtures for route warming, first-party server error detection, and uncaught page error checks
- Added Better Auth organization hooks that enforce workspace domain and team invitation policy even for raw plugin calls
- Added regression tests for local automation auth, cleanup behavior, organization hooks, and E2E support utilities

## Included in this release

### Local automation auth

- Added `LOCAL_AUTOMATION_AUTH_ENABLED` as the explicit feature flag for local automation sign-in
- Kept the automation flow disabled in production even when the flag is present
- Enabled Better Auth email/password sign-up with auto sign-in only while local automation auth is enabled
- Added `POST /api/local-auth/scenario` for creating a Better Auth credential user and returning the active session cookie
- Added `DELETE /api/local-auth/scenario` for deleting the current automation user and cleaning up now-memberless local organizations
- Added generated automation credentials with `local-agent+...@local-agent.test` email addresses and strong random passwords
- Allowed explicit automation credentials for deterministic scenarios while rejecting non-automation email domains
- Retried generated sign-up credentials when Better Auth reports a duplicate generated email
- Rejected malformed JSON and invalid request shapes before calling Better Auth
- Forwarded Better Auth response cookies from create and cleanup responses so browser contexts keep the real session state
- Revalidated the root layout after automation cleanup
- Added local automation account helpers and repository functions under the accounts feature slice
- Added a login-page automation panel that creates a local user, refreshes the router, and navigates to the sanitized redirect
- Wrapped the automation panel in a local Suspense boundary so the login route stays compatible with search params
- Documented the local automation flow in `AGENTS.md` for browser automation agents

## Upgrade notes

Compared with `v0.0.7`, template users should note:

- No Prisma migration is required for this release
- Local automation auth is intended for development and browser automation only; never enable `LOCAL_AUTOMATION_AUTH_ENABLED` in production
- `.env.example` includes `LOCAL_AUTOMATION_AUTH_ENABLED=true` for local template usage
- When using Playwright locally for the first time, run `npm run e2e:install` to install Chromium
- `npm run e2e` starts the Next.js dev server automatically unless `PLAYWRIGHT_START_SERVER=false`
- The default Playwright base URL is `http://127.0.0.1:3127`; override it with `PLAYWRIGHT_BASE_URL`
- Playwright artifacts are written to `playwright-report`, `test-results/playwright`, and `blob-report`, all ignored by Git
- Jest now ignores the `e2e/` folder so browser specs do not run in the unit test suite
- Better Auth organization invitations are now validated by plugin hooks as well as feature server actions
- Direct Better Auth invitation calls that bypass feature actions must satisfy workspace allowed-domain and team ownership policy
- Local automation cleanup deletes only the current automation user and organizations that become memberless after that user is removed
