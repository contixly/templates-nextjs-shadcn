# v0.0.11

Documentation system and localized public documentation release for the Next.js application template.

This release builds on `v0.0.10` by turning `/docs` into a maintained public
documentation center for the template. It adds a full documents-system runtime, localized
Markdown/MDX content, documentation search, release and changelog pages, and OpenSpec-backed
requirements for the documentation surface.

## Highlights

- Added a public `/docs` documentation system with a shared shell, sidebar, breadcrumb, page actions,
  table of contents, theme controls, canonical routes, and generated share images
- Added locale-aware Markdown and MDX document resolution so English and Russian content variants
  share stable canonical URLs and deterministic fallback behavior
- Published bilingual documentation for account management, API access, application settings,
  workspace flows, developer guidance, release history, and weekly change summaries
- Added `/api/v1/documents-system/search` for locale-aware page and heading search
- Added MDX rendering components for callouts, tabs, link grids, code-copy controls, heading anchors,
  local images, and safe internal links
- Added documents-system OpenSpec requirements and focused Jest coverage for registry, navigation,
  metadata, search, table of contents, and link validation behavior

## Included in this release

### Public documentation runtime

- Added public App Router routes for `/docs`, `/docs/[...slug]`, `/docs/og/[...slug]`, and static
  documentation metadata images
- Added a shared documentation layout with sidebar navigation, page breadcrumb, search command, home
  navigation, and theme controls
- Added page-level previous and next navigation derived from sorted visible documents
- Added metadata handling for titles, descriptions, edited dates, reading time, page status, and
  generated share images
- Added production and local visibility rules for `published`, `archived`, `draft`, `review`, and
  hidden documentation pages

### Localized content registry

- Added a registry for local Markdown and MDX content under
  `src/features/documents-system/content/`
- Added `.en.md`, `.en.mdx`, `.ru.md`, and `.ru.mdx` locale suffix support while keeping canonical
  public URLs free of locale suffixes
- Added deterministic fallback behavior when a page is missing in the selected interface locale
- Added fallback language markers so readers can see when a page is rendered in a different content
  language
- Added duplicate-locale, unsupported-locale, and locale-suffixed-static-param validation

### Search and MDX rendering

- Added `/api/v1/documents-system/search` with bounded query length, stable empty responses, and
  locale selection
- Indexed visible document metadata and source headings for page and section results
- Added exact-token fallback for short queries, multi-word ranking, fuzzy matching, and keyboard
  layout normalization
- Added heading extraction that ignores fenced code blocks, including tilde-fenced examples
- Added MDX components for callouts, tabs, link grids, code blocks, copy buttons, heading anchors,
  local images, and internal document links
- Disabled broken or unpublished internal links in production while keeping local authoring feedback
  visible

### Public content and navigation

- Added localized documentation pages for account settings, profile connections, session security,
  account deletion, API keys, `/api/v1`, permissions, application caching, localization, OAuth
  providers, runtime security, settings shell, developer workflows, workspace flows, glossary, and
  quick start guidance
- Added public release pages for `v0.0.1` through `v0.0.10`
- Added weekly changelog pages for April, May, and July 2026 history windows
- Added public documentation links to the application header and secondary navigation
- Added localized documents-system message catalogs and message loading

### Documentation and specs

- Added `openspec/specs/documents-system/spec.md` covering public routes, shell behavior, registry
  metadata, locale variants, visibility rules, navigation, static params, MDX rendering, page
  metadata, internal links, search, metadata images, and caching modes
- Added a documents content localization design note under `docs/superpowers/specs/`
- Updated agent and OpenSpec guidance so future public documentation work stays aligned with the
  documents-system content structure
- Updated template package metadata to `0.0.11` in `package.json` and `package-lock.json`

## Quality and verification

## Upgrade notes

Compared with `v0.0.10`, template users should note:

- No Prisma migration is required for this release.
- The template package metadata now reports `0.0.11`.
- The template now depends on `@next/mdx`, `@mdx-js/loader`, `@mdx-js/react`, `@types/mdx`, `glob`,
  `gray-matter`, `remark-frontmatter`, `remark-gfm`, and `sharp` for local documentation content and
  generated documentation images.
- `next.config.ts` now enables `.md` and `.mdx` page extensions and wraps the Next.js config with
  `@next/mdx`.
- A root `mdx-components.tsx` file is required by the Next.js App Router MDX integration and is now
  included.
- Public documentation content lives under `src/features/documents-system/content/`; localized
  variants should use `.en` and `.ru` filename suffixes while keeping internal links canonical under
  `/docs/...`.
- `/api/v1/documents-system/search` is a public documentation search endpoint and does not require a
  browser session.
- Public documentation pages are generated from local content, so broken internal documentation
  links and duplicate locale variants should be fixed before publishing.
