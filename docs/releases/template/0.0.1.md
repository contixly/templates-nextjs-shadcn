# v0.0.1

Initial public release of the Next.js application template.

This release captures the `main` branch state published at commit `35f7126` and establishes the first reusable baseline for building new services on top of Next.js, Prisma, Better Auth, Tailwind CSS v4, and shadcn/ui.

## Highlights

- Next.js 16 + React 19 + TypeScript starter with App Router structure
- Tailwind CSS v4 and shadcn/ui component foundation
- Prisma + PostgreSQL setup with generated client and initial migration
- Better Auth integration with social login scaffolding
- Feature Slice Design structure for organizing business logic
- Shared layout, navigation, metadata, sitemap, robots, Open Graph, and Twitter image setup
- Example authenticated application shell with dashboard, workspace, and account areas

## Included in this release

### Application foundation

- Public landing page and authenticated welcome flow
- Protected and public route handling through `proxy.ts`
- Health check endpoint at `/api/health`
- Shared metadata helpers and app-level SEO defaults
- Reusable UI primitives and application shell components

### Authentication and accounts

- Better Auth wired to Prisma
- Social auth scaffolding for Google, GitHub, GitLab, VK, and Yandex OAuth
- Session-aware protected routing
- Automatic workspace creation after successful sign-in
- Account profile, connected accounts, session management, and account deletion flows

### Workspaces and example product structure

- Workspace repository and server action patterns
- Workspace create, update, delete, and load flows
- Example dashboard page and workspace-aware navigation
- Feature-based route definitions and typed page metadata

### Developer experience

- README updated for template usage
- `TEMPLATE.md` added with post-generation setup checklist
- `.env.example` added for initial environment configuration
- `CONTRIBUTING.md` and `SECURITY.md` added
- Jest test coverage for core UI and configuration regressions

## Fixes and refinements included before 0.0.1

- Upgraded Better Auth dependencies to `v1.6.2`

- Added Prisma config fallback for local `DATABASE_URL` usage in tooling
- Removed placeholder workspace counts and simplified workspace data flow
- Streamlined workspace UI and repository logic
- Improved `ThemeSwitcher` hydration behavior and added fallback rendering tests
- Refined landing page copy and template messaging to better reflect the repository's purpose

## Notes

Before using this template for a real project, configure:

- `DATABASE_URL`
- `BETTER_AUTH_SECRET`
- `BETTER_AUTH_URL`
- `PUBLIC_BASE_URL`
- OAuth provider credentials for the providers you plan to enable

After generating a new repository from this template, follow `TEMPLATE.md` for the recommended setup sequence.
