# v0.0.3

Organization-backed workspace management release for the Next.js application template.

This release builds on `v0.0.2` by replacing the earlier workspace-only foundation with Better Auth organizations as the
backing collaboration model. It adds organization-scoped routing, workspace settings, invitations, member management,
role-aware permissions, and stronger onboarding flows for users who do not yet belong to a workspace.

## Highlights

- Added Better Auth organization support while preserving "Workspace" as the product-facing term
- Introduced organization-scoped routes under `/:organizationKey/...` with slug-preferred URLs
- Added active and fallback organization resolution for dashboard navigation
- Added dedicated workspace settings pages for workspace details, users, invitations, teams, and roles
- Added workspace invitation creation, listing, link copying, accept, reject, and pending-invitation entry points
- Added member directory, direct member add, and built-in owner/admin/member role management
- Added zero-workspace onboarding and invitation review paths
- Refreshed the public landing copy and README to describe the organization, invitation, and role model

## Included in this release

### Organization-backed workspace model

- Better Auth organization plugin wired into the auth configuration
- Prisma schema updated with organization, member, invitation, and active organization session fields
- Organization repositories added with cache tags for accessible organizations and organization members
- Workspace types preserved as product-facing aliases while data access resolves through organizations
- Workspace context handling added through active organization sessions and deterministic fallback

### Organization-scoped routing

- Global `/dashboard` now resolves the current user's active organization or deterministic fallback
- Workspace routes now live under `/:organizationKey`, where the key can be an organization slug or id
- Organization root routes redirect to the slug-preferred workspace dashboard after validating access
- Sidebar workspace labels and breadcrumbs resolve from the URL organization context
- Route-level loading and forbidden-state behavior added for organization-scoped pages

### Workspace settings surfaces

- Shared settings shell added for account and workspace settings
- Workspace settings navigation added for workspace details, users, invitations, teams, and roles
- Workspace details form supports name, slug, and guarded delete behavior
- Teams and roles pages provide placeholder surfaces for future expansion
- Account settings pages were aligned with the shared settings layout and section composition

### Members and roles

- Workspace users page now renders organization members with identity, role labels, and joined-at details
- Authorized users can add existing users to a workspace by user id
- Authorized users can update another member's role from the users table
- Built-in role rules cover owner, admin, and member assignment constraints
- Regular members retain read-only access to the workspace directory when they do not have management permissions

### Invitations

- Workspace invitations settings page lists organization invitations with recipient, role, inviter, dates, and status
- Authorized users can create invitations with role selection and copy shareable invitation links
- Invitation decision route lets authenticated invitees review, accept, or reject an invitation
- Acceptance requires the invitation email to match the user's verified primary email
- Personal pending-invitation surfaces were added under account settings and the welcome flow
- Pending invitations past their expiration are shown as expired in the UI

### Onboarding and navigation

- Users with no accessible workspaces are sent to welcome instead of dead-end organization pages
- Welcome page exposes workspace creation and pending invitation review
- Workspace switchers update active organization context through explicit user actions
- Main navigation and workspace cards were updated for organization-backed routes

### Documentation and tooling

- README and public marketing blocks now describe organization-backed workspaces, invitations, roles, and settings
- OpenSpec project configuration and archived specs were added for the organization-management workstream
- Jest configuration was expanded to cover the new folder structure and feature tests
- Skills and agent tooling were updated for the repository workflow

## Quality and verification

- Added tests for organization route guards, slug generation, compatibility helpers, and repository cache tags
- Added tests for workspace creation, updates, deletion, member management, role assignment, and invitations
- Added tests for workspace settings pages, settings navigation, onboarding guard, and sidebar switcher behavior
- Added tests for account invitations and shared settings shell composition
- Added regression coverage for localized workspace/account copy and page metadata consumers

## Upgrade notes

Compared with `v0.0.2`, template users should note:

- Workspaces are now backed by Better Auth organizations instead of a standalone workspace model
- Existing deployments should run the new Prisma migrations before using the organization and invitation flows
- Workspace URLs now prefer `/:organizationKey/...` routes, where `organizationKey` is the organization slug when
  available and id otherwise
- Invitation acceptance depends on the invitee having a verified primary email that matches the invited address
- Workspace management permissions now follow Better Auth organization permission checks and built-in owner/admin/member
  assignment rules
