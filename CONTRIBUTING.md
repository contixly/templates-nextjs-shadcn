# Contributing to this template

This repository is a **starter template** for new Next.js projects. Contributions help everyone who generates a new repo from it.

## What we welcome

- Bug fixes and small improvements that keep the template easy to clone and run.
- Documentation fixes (README, `TEMPLATE.md`, comments) that reduce setup friction.
- Dependency updates that stay aligned with the stack described in `README.md` and `AGENTS.md`.

## What to avoid

- Large opinionated refactors that are not clearly useful for all consumers of the template.
- Features that require heavy third-party services without an opt-in path and documentation.

## How to contribute

1. Open an issue (or discussion, if enabled) to describe the change for anything non-trivial.
2. Use a branch named `fix/`, `feat/`, or `chore/` per project conventions.
3. Run `npm run lint` and `npm run test` before opening a pull request.
4. Keep commits in [Conventional Commits](https://www.conventionalcommits.org/) style when possible.

## Publishing the template on GitHub

Maintainers: in the repository **Settings → General**, enable **Template repository** so the green **Use this template** button appears for visitors.
