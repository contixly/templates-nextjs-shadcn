# Быстрый старт

Эта страница нужна, когда вы создаете новый сервис на основе шаблона. Она описывает минимальный
путь от созданного репозитория до локально работающего приложения с авторизацией, базой данных и
сценариями рабочих пространств.

## Что понадобится

- Node.js 22 или новее.
- PostgreSQL база для Prisma.
- Redis или Valkey опционально, если нужен распределенный кеш для Cache Components и ISR.
- OAuth-приложения для провайдеров, которые должны появиться на странице входа.

## Создайте репозиторий

Создайте репозиторий из шаблона GitHub, затем установите зависимости:

```bash
npm install
```

Перед публикацией сервиса обновите имя пакета и видимые метаданные нового проекта. Полный список
первичных действий находится в `TEMPLATE.md`.

## Настройте переменные окружения

Скопируйте значения из `.env.example` или README в `.env.local`. Для локального запуска нужны:

| Переменная | Назначение |
| ---------- | ---------- |
| `DATABASE_URL` | Подключение PostgreSQL, которое использует Prisma. |
| `BETTER_AUTH_SECRET` | Секрет для подписи Better Auth сессий. |
| `BETTER_AUTH_URL` | Серверный базовый адрес приложения, например `http://localhost:3000`. |
| `NEXT_PUBLIC_APP_BASE_URL` | Публичный базовый адрес приложения для браузерного кода. |
| `PUBLIC_DEFAULT_LOCALE` | Локаль интерфейса и документации по умолчанию. При ошибке используется `en`. |

OAuth-переменные задавайте только для провайдеров, которые реально используются. Провайдеры с
неполным набором учетных данных не показываются на страницах входа и подключений.

## Подготовьте базу

После настройки `DATABASE_URL` примените миграции Prisma:

```bash
npx prisma migrate dev
```

Если нужно посмотреть локальных пользователей, сессии, организации, команды или API-ключи,
используйте `npx prisma studio`.

## Запустите приложение

Запустите сервер разработки:

```bash
npm run dev
```

Откройте `/`, чтобы проверить публичную страницу, `/auth/login`, чтобы войти, и `/docs`, чтобы
прочитать публичную документацию. После входа сценарии рабочих пространств помогут создать рабочее
пространство или посмотреть ожидающие приглашения.

## Проверьте поверхность шаблона

Перед началом продуктовой разработки запустите релевантные проверки:

```bash
npm run lint
npm run test
npm run e2e
```

E2E-команда по умолчанию сама запускает локальный сервер и включает флаги локальной авторизации для
автоматизации, которые нужны браузерным сценариям.
