# Глоссарий

Этот глоссарий описывает термины, которые встречаются в шаблоне и его документации. Он не является
словарем конкретного продукта: после создания собственного приложения его можно расширить или
заменить предметными терминами вашего домена.

## Приложение и маршруты

| Термин | Значение |
| ------ | -------- |
| Шаблон | Стартовый Next.js проект с авторизацией, рабочими пространствами, локализацией, API-ключами, документацией и проверками качества. |
| App Router | Маршрутизация Next.js, в которой страницы, макеты и обработчики маршрутов живут в `src/app`. |
| Публичный маршрут | Маршрут, доступный без входа в аккаунт, например `/`, `/auth/login`, `/docs` или `/api/v1`. |
| Защищенный маршрут | Маршрут, который требует действующую сессию Better Auth. |
| Прокси | Файл `src/proxy.ts`, который проверяет границу публичных и защищенных маршрутов до рендера страницы. |
| Функциональный срез | Подход к организации кода, где бизнес-логика отдельной возможности лежит в `src/features/{feature}`. |
| Серверное действие | Серверная мутация, которая валидирует входные данные, проверяет доступ, меняет данные и обновляет кеш. |
| Оболочка приложения | Защищенный макет и навигация для страниц приложения после входа. |
| Панель | Стартовая защищенная обзорная страница, которая показывает оболочку и должна быть заменена продуктовым контентом. |

## Аккаунты и авторизация

| Термин | Значение |
| ------ | -------- |
| Better Auth | Библиотека авторизации, которая управляет пользователями, сессиями, OAuth-провайдерами, организациями, командами и API-ключами. |
| Сессия | Подтвержденный вход пользователя в приложение. Сессия используется для доступа к защищенным маршрутам и серверным действиям. |
| OAuth-провайдер | Внешний способ входа, например Google, GitHub, GitLab, VK или Yandex. В интерфейсе показываются только полностью настроенные провайдеры. |
| Подключенный аккаунт | Связь текущего пользователя с OAuth-провайдером на странице настроек аккаунта. |
| Локальная авторизация для автоматизации | Локальный режим для Playwright и браузерной автоматизации. Он создает временного пользователя только вне боевого окружения и только при явном включении. |
| Опасные действия | Раздел аккаунта для необратимых действий, например удаления текущего пользователя. |

## Рабочие пространства

| Термин | Значение |
| ------ | -------- |
| Рабочее пространство | Пользовательское название рабочей области приложения. Внутри шаблона рабочее пространство хранится как организация Better Auth. |
| Организация | Техническая модель Better Auth, которая хранит участников, роли, команды, приглашения и API-ключи рабочего пространства. |
| Ключ организации | Часть URL в маршрутах `/w/:organizationKey/...`. Это может быть slug организации или ее id. |
| Активное рабочее пространство | Рабочее пространство, выбранное в текущей сессии пользователя. `/dashboard` перенаправляет в него или в стабильный запасной вариант. |
| Участник | Пользователь, который состоит в рабочем пространстве и имеет одну из встроенных ролей. |
| Роль | Набор прав участника. В шаблоне есть owner, admin и member. |
| Приглашение | Запрос присоединиться к рабочему пространству. Приглашение можно принять или отклонить после входа в аккаунт. |
| Команда | Явная подгруппа внутри рабочего пространства. Новое рабочее пространство не создает команду автоматически; ноль команд считается нормальным состоянием. |
| Приглашение в команду | Приглашение, которое после принятия добавляет пользователя в рабочее пространство и выбранную команду. |
| Списки разрешенных доменов email | Ограничение рабочего пространства, которое разрешает приглашения только на указанные домены email и предупреждает о текущих участниках вне политики. |
| Онбординг без рабочего пространства | Экран для пользователя без доступных рабочих пространств. Он оставляет доступными создание рабочего пространства и просмотр личных приглашений. |
| Настройки рабочего пространства | Отдельные страницы разделов для деталей рабочего пространства, участников, приглашений, команд, ролей и API-ключей рабочего пространства. |

## API и интеграции

| Термин | Значение |
| ------ | -------- |
| API-ключ | Ключ для доступа интеграций к `/api/v1`. Браузерные cookies не используются для этих маршрутов. |
| API v1 | Стартовая внешняя API-поверхность, которая авторизуется API-ключами и возвращает стабильный JSON-формат. |
| Персональный API-ключ | Ключ пользователя. Он действует от имени владельца и читает данные рабочего пространства только пока пользователь остается участником рабочего пространства. |
| API-ключ организации | Ключ рабочего пространства. Он действует от имени одной организации и управляется в настройках рабочего пространства. |
| Набор прав | Готовая группа областей доступа для API-ключа. Продукт может расширять эти наборы собственными правами. |
| Область доступа | Конкретное право API-ключа, которое проверяется перед доступом к обработчику маршрута. |
| Ограничение частоты | Ограничение частоты запросов для API-ключа. |
| Формат JSON-ответа | Единый формат ответа API: `{ data: ... }` для успеха и `{ error: { code, message } }` для обработанной ошибки. |

## Локализация, кеш и документация

| Термин | Значение |
| ------ | -------- |
| `PUBLIC_DEFAULT_LOCALE` | Переменная окружения, которая задает локаль по умолчанию для приложения и документации. Если значение не поддерживается, используется `en`. |
| Вариант локали | Файл документации с суффиксом локали, например `index.en.md` или `index.ru.md`. |
| Языковой маркер запасного контента | Маркер в UI документации, который показывает, что текущая страница открыта не на выбранной локали интерфейса. |
| Cache Components | Режим Next.js, используемый шаблоном для серверного кеширования. |
| Удаленный кеш | Опциональный Redis/Valkey кеш для Cache Components и ISR, включаемый через `REMOTE_CACHING_ENABLED=true`. |
| OpenSpec | Набор требований в `openspec/specs`, по которым проверяются пользовательские сценарии и E2E-покрытие. |
| E2E на базе OpenSpec | Сценарии Playwright в `e2e/specs/<capability>`, которые зеркалят возможность OpenSpec. |
| Оболочка настроек | Общий паттерн макета для страниц настроек аккаунта и рабочего пространства. |
