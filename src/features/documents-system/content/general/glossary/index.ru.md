# Глоссарий

Этот глоссарий описывает термины, которые встречаются в шаблоне и его документации. Он не является
словарем конкретного продукта: после создания собственного приложения его можно расширить или
заменить предметными терминами вашего домена.

## Приложение и маршруты

| Термин | Значение |
| ------ | -------- |
| Шаблон | Стартовый Next.js проект с авторизацией, рабочими пространствами, локализацией, API-ключами, документацией и проверками качества. |
| App Router | Маршрутизация Next.js, в которой страницы, layouts и route handlers живут в `src/app`. |
| Public route | Маршрут, доступный без входа в аккаунт, например `/`, `/login`, `/docs` или `/api/v1`. |
| Protected route | Маршрут, который требует действующую Better Auth сессию. |
| Proxy | Файл `src/proxy.ts`, который проверяет границу публичных и защищенных маршрутов до рендера страницы. |
| Feature Slice | Подход к организации кода, где бизнес-логика отдельной возможности лежит в `src/features/{feature}`. |
| Server action | Серверная мутация, которая валидирует входные данные, проверяет доступ, меняет данные и обновляет кеш. |

## Аккаунты и авторизация

| Термин | Значение |
| ------ | -------- |
| Better Auth | Библиотека авторизации, которая управляет пользователями, сессиями, OAuth-провайдерами, организациями, командами и API-ключами. |
| Сессия | Подтвержденный вход пользователя в приложение. Сессия используется для доступа к protected routes и server actions. |
| OAuth-провайдер | Внешний способ входа, например Google, GitHub, GitLab, VK или Yandex. В интерфейсе показываются только полностью настроенные провайдеры. |
| Подключенный аккаунт | Связь текущего пользователя с OAuth-провайдером на странице настроек аккаунта. |
| Local automation auth | Локальный режим для Playwright и browser automation. Он создает временного пользователя только вне production и только при явном включении. |

## Рабочие пространства

| Термин | Значение |
| ------ | -------- |
| Workspace | Пользовательское название рабочей области приложения. Внутри шаблона workspace хранится как Better Auth Organization. |
| Organization | Техническая модель Better Auth, которая хранит участников, роли, команды, приглашения и API-ключи workspace. |
| Organization key | Часть URL в маршрутах `/w/:organizationKey/...`. Это может быть slug организации или ее id. |
| Active workspace | Workspace, который выбран в текущей сессии пользователя. `/dashboard` перенаправляет в активный workspace или в стабильный fallback. |
| Участник | Пользователь, который состоит в workspace и имеет одну из встроенных ролей. |
| Роль | Набор прав участника. В шаблоне есть owner, admin и member. |
| Приглашение | Запрос присоединиться к workspace. Приглашение можно принять или отклонить после входа в аккаунт. |
| Team | Явная подгруппа внутри workspace. Новый workspace не создает team автоматически; ноль команд считается нормальным состоянием. |
| Allowed email domains | Ограничение workspace, которое разрешает приглашения только на указанные email-домены и предупреждает о текущих участниках вне политики. |
| Zero-workspace onboarding | Экран для пользователя без доступных workspace. Он оставляет доступными создание workspace и просмотр личных приглашений. |

## API и интеграции

| Термин | Значение |
| ------ | -------- |
| API key | Ключ для машинного доступа к `/api/v1`. Браузерные cookies не используются для этих маршрутов. |
| Personal API key | Ключ пользователя. Он действует от имени владельца и читает workspace-данные только пока пользователь остается участником workspace. |
| Organization API key | Ключ workspace. Он действует от имени одной организации и управляется в настройках workspace. |
| Permission preset | Готовый набор scopes для API-ключа. Продукт может расширять эти presets собственными правами. |
| Scope | Конкретное право API-ключа, которое проверяется перед доступом к route handler. |
| Rate limit | Ограничение частоты запросов для API-ключа. |
| JSON envelope | Единый формат ответа API: `{ data: ... }` для успеха и `{ error: { code, message } }` для обработанной ошибки. |

## Локализация, кеш и документация

| Термин | Значение |
| ------ | -------- |
| `PUBLIC_DEFAULT_LOCALE` | Переменная окружения, которая задает локаль по умолчанию для приложения и документации. Если значение не поддерживается, используется `en`. |
| Locale variant | Файл документации с суффиксом локали, например `index.en.md` или `index.ru.md`. |
| Fallback language marker | Маркер в UI документации, который показывает, что текущая страница открыта не на выбранной локали интерфейса. |
| Cache Components | Режим Next.js, используемый шаблоном для серверного кеширования. |
| Remote cache | Опциональный Redis/Valkey кеш для Cache Components и ISR, включаемый через `REMOTE_CACHING_ENABLED=true`. |
| OpenSpec | Набор требований в `openspec/specs`, по которым проверяются пользовательские сценарии и E2E-покрытие. |
