# Глоссарий

Глоссарий фиксирует демонстрационные термины из скопированного набора страниц. В продукте этот
раздел можно заменить собственным словарём, сохранив формат таблиц и группировку по темам.

## Подготовка программы: активности

| Термин              | Значение                                                                                                                                                                                |
|---------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Активность          | Единица программы мероприятия: доклад, встреча или другой программный элемент, который отображается в списке активностей и может быть добавлен в расписание.                            |
| Карточка активности | Страница или форма с подробными данными активности: названием, описанием, участниками, материалами, статусами и настройками публикации.                                                 |
| Участник активности | Человек или служебная запись, связанная с активностью в определенной роли, например спикер.                                                                                             |
| Спикер активности   | Участник активности с ролью спикера. Для быстро созданной активности это основной участник, которого пользователь выбирает или создает в форме.                                         |
| Контрибьютор        | Профиль человека в рабочем пространстве. Контрибьютор может быть выбран при добавлении участника команды или как существующий спикер активности.                                        |
| Служебный спикер    | Техническая запись, которую система использует, когда активность создается без реального спикера. Ее нужно заменить на настоящего участника, если активность публикуется с докладчиком. |
| Название доклада    | Короткое название активности, по которому ее ищут в таблице активностей и распознают в программе.                                                                                       |
| Заявка              | Объект, связанный с активностью и ее прохождением через программный процесс. При быстром создании заявка создается автоматически в начальном статусе.                                   |

## Подготовка программы: отбор заявок

| Термин                         | Значение                                                                                                                                                                          |
|--------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Отбор заявок                   | Рабочий процесс, в котором программный комитет рассматривает заявки мероприятия, назначает ответственных, оставляет оценки и готовит решение по программе.                         |
| Панель голосования             | Экран **Отбор заявок** с таблицей заявок, быстрыми фильтрами, рейтингом, голосами, назначениями кураторов и ревьюеров, материалами и внутренними комментариями.                    |
| Программный комитет, ПК         | Команда участников, которые оценивают заявки и помогают сформировать программу мероприятия.                                                                                        |
| Куратор заявки                 | Участник ПК, отвечающий за сопровождение конкретной заявки. Куратор отображается в колонке **Кураторы** и может быть основным, если стоит первым в списке текущих кураторов.       |
| Основной куратор               | Первый текущий куратор заявки после сортировки назначений. В окне кураторов и ревьюеров помечается как основной.                                                                  |
| Ревьюер заявки                 | Участник ПК, назначенный для экспертной оценки заявки. Ревьюер оставляет голос с решением, числовой оценкой и комментарием.                                                       |
| Голос                          | Оценка заявки конкретным участником ПК. Содержит решение, числовую оценку и комментарий.                                                                                          |
| Резолюция голоса               | Итоговая рекомендация в голосе: **Принять**, **Отклонить** или **Воздержаться**.                                                                                                  |
| Рейтинг заявки                 | Средняя числовая оценка заявки по видимым голосам. В панели может быть скрыт правилами видимости оценок.                                                                         |
| Открытое голосование           | Состояние заявки, в котором участники ПК могут создать или изменить свой голос. В панели отбора к нему относятся заявки до попадания в программу.                                  |
| Внутренний статус заявки       | Рабочий статус заявки, который команда использует для отбора: например разбор, проработка, ожидание материалов или попадание в программу. Отображается в колонке **Статус**.       |
| Системный статус заявки        | Статус заявки в SRM. Доступен как скрытая по умолчанию колонка **Системный статус**.                                                                                              |
| Назначение                     | Связь участника команды с заявкой в роли куратора или ревьюера. Текущими считаются активные и завершенные назначения; закрытые назначения не участвуют в текущем отборе.           |
| Быстрый фильтр                 | Готовый рабочий срез панели отбора, например **Без голосов**, **Без куратора**, **Ждет моей оценки** или **Я оценил**.                                                            |
| Материалы заявки               | Ссылки или файлы, прикрепленные к заявке для оценки программным комитетом.                                                                                                        |
| Внутренний комментарий заявки  | Комментарий команды по заявке, который используется для обсуждения внутри программного комитета и отображается в колонке **Последний комментарий**.                                |
| Видимость оценок               | Правила, которые определяют, когда пользователь видит средний рейтинг и детали чужих голосов. Зависят от роли и настроек текущей сессии браузера.                                  |
| Доклад вне критериев оценки    | Заявка, которая уже не участвует в текущем голосовании из-за системного статуса. Такие доклады можно скрывать или показывать отдельным переключателем в панели отбора.             |

## Подготовка программы: команда

| Термин                    | Значение                                                                                                                                                                               |
|---------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Команда пространства      | Участники, которым доступ назначен на уровне рабочего пространства. Такой доступ может распространяться на проекты и мероприятия внутри пространства.                                  |
| Команда проекта           | Участники, которым доступ назначен на уровне проекта. В списке проекта также могут отображаться участники пространства, если их роли действуют сверху вниз.                           |
| Команда мероприятия       | Участники, которые могут работать с конкретным мероприятием через доступ к пространству, проекту или самому мероприятию.                                                               |
| Уровень команды           | Контекст, где участнику назначен или из которого пришел доступ: пространство, проект или мероприятие. Уровень помогает понять, где можно управлять участником.                         |
| Унаследованный доступ     | Доступ, который приходит с верхнего уровня команды. Например, участник пространства может быть виден в проекте или мероприятии без повторного добавления.                              |
| Ограниченный уровень      | Уровень команды, который текущий пользователь не может просматривать или изменять из-за своих прав. Это не означает, что на уровне нет участников.                                     |
| Роль участника команды    | Назначенная участнику роль, которая определяет его права на выбранном уровне. У участника может быть несколько ролей.                                                                  |
| Группа ролей              | Объединение близких ролей по направлению работы: управление, программа, партнёры, поддержка, трансляция, маркетинг или наблюдатели.                                                   |
| Основная роль             | Роль, которая первой показывается в компактном списке команды. Она помогает быстро понять основное участие человека, но не заменяет полный список ролей.                               |
| Приглашение участника     | Добавление человека в команду: выбор существующего контрибьютора, создание нового участника или массовое приглашение по email.                                                        |
| Заблокированный участник  | Участник, который остается в данных команды, но скрыт из активного списка по умолчанию.                                                                                                |
| Восстановление участника  | Возврат заблокированного участника в активный список на том уровне команды, где доступ нужно вернуть.                                                                                  |

## Подготовка программы: обмен и импорт

| Термин                       | Значение                                                                                                                                                 |
|------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------|
| Импорт обмена                | Процесс создания или загрузки активностей через механизм обмена данными. Используется как для XLS-импорта, так и для быстрого создания одной активности. |
| Пайплайн                     | Серверный процесс обработки операции. В быстром создании пайплайн создает активность и возвращает статус выполнения.                                     |
| Инстанс пайплайна            | Конкретный запуск пайплайна с собственным идентификатором и статусом. Этот идентификатор полезен при разборе ошибок.                                     |
| Быстрое заведение активности | Сценарий создания одной активности с минимальным набором полей: названием доклада и способом заполнения основного спикера.                               |
