# Локализованный контент документации

Контент документации локализуется через имя файла. URL остается каноническим и не содержит локаль.

## Имена файлов

Для публичных страниц используйте явные суффиксы локали:

- `page.en.md` или `page.en.mdx`;
- `page.ru.md` или `page.ru.mdx`.

Для индексных страниц используйте `index.en.md` и `index.ru.md`. Оба файла публикуются по одному
каноническому URL.

## Канонические URL

Суффиксы локали удаляются из URL. Например:

| Файл | URL |
| ---- | --- |
| `workspace/settings.en.md` | `/docs/workspace/settings` |
| `workspace/settings.ru.md` | `/docs/workspace/settings` |

Внутренние ссылки всегда должны использовать канонические URL `/docs/...` без `.en` или `.ru`.

## Запасной контент

Если страница существует только в одной локали, система документации может использовать ее как
запасной контент для обоих языков интерфейса. UI показывает языковой маркер, когда отображается
запасной контент.

Используйте запасной контент осознанно. Публичная документация шаблона обычно должна иметь оба
варианта: `en` и `ru`.

## Валидация

Реестр контента отклоняет дублирующиеся файлы для одного канонического URL и локали,
неподдерживаемые суффиксы локалей, статические параметры с суффиксом локали и сломанные внутренние
ссылки.
