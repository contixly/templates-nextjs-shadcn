# v0.0.9

v0.0.9 добавляет в шаблон управление API-ключами и scoped machine access. В релиз входят
personal и organization keys, стартовый `/api/v1` surface и settings UI для создания, изменения,
отключения и удаления ключей.

## Главное

- Personal API keys управляются на `/user/api-keys`.
- Organization API keys управляются на `/w/:organizationKey/settings/api-keys`.
- API-клиенты проходят аутентификацию через заголовок `x-api-key`, а не через browser session
  cookies.
- Ключи поддерживают одноразовый показ секрета, permission presets, expiration и rate limits.
- Стартовые `/api/v1` endpoints возвращают key metadata, organizations, members, teams и team
  members.
- Workspace роли owner и admin могут управлять organization keys по умолчанию.

## Что изменилось для пользователей шаблона

Personal key действует от имени владельца и читает organization data только пока этот пользователь
остается участником workspace. Organization key действует от имени одной workspace organization и
остается ограниченным этой organization. Оба семейства ключей используют явные permissions, поэтому
product teams могут добавлять собственные API routes и permission presets без передачи сырых Better
Auth records в client code.

Страницы управления показывают поясняющий текст, create и edit dialogs, status metadata, key
prefix, scopes, rate-limit values, expiration state, last-use state и delete confirmation.

## Upgrade notes

- Перед использованием API key features нужна Prisma migration.
- После установки зависимостей нужно сгенерировать Prisma Client, чтобы был доступен
  `prisma.apiKey`.
- API-клиенты должны отправлять ключи в заголовке `x-api-key`.
- Секрет ключа показывается только один раз при создании.
- Обычное сохранение edit form не продлевает expiration; renewal включается явно.
- Рекомендуется Node.js 22+, потому что текущая cache handler dependency требует эту версию.

## Качество

Релиз добавляет OpenSpec requirements, Jest-покрытие для API key configuration, permissions,
schemas, actions, route handlers, management helpers и UI behavior, а также Playwright-проверки для
personal keys, organization keys и organization permission boundaries.
