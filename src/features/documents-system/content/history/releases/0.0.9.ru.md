# v0.0.9

v0.0.9 добавляет в шаблон управление API-ключами и машинный доступ по областям доступа. В релиз
входят персональные ключи и ключи организаций, стартовая поверхность `/api/v1` и UI настроек для
создания, изменения, отключения и удаления ключей.

## Главное

- Персональные API-ключи управляются на `/user/api-keys`.
- API-ключи организаций управляются на `/w/:organizationKey/settings/api-keys`.
- API-клиенты проходят аутентификацию через заголовок `x-api-key`, а не через cookie браузерной сессии.
- Ключи поддерживают одноразовый показ секрета, наборы прав, срок действия и ограничения частоты.
- Стартовые маршруты `/api/v1` возвращают метаданные ключа, организации, участников, команды и участников команд.
- Роли рабочего пространства `owner` и `admin` могут управлять ключами организации по умолчанию.

## Что изменилось для пользователей шаблона

Персональный ключ действует от имени владельца и читает данные организации только пока этот
пользователь остается участником рабочего пространства. Ключ организации действует от имени одной
организации рабочего пространства и остается ограниченным этой организацией. Оба семейства ключей
используют явные права, поэтому продуктовые команды могут добавлять собственные маршруты API и
наборы прав без передачи сырых записей Better Auth в клиентский код.

Страницы управления показывают поясняющий текст, диалоги создания и редактирования, метаданные
статуса, префикс ключа, области доступа, значения ограничений частоты, состояние срока действия,
состояние последнего использования и подтверждение удаления.

## Заметки по обновлению

- Перед использованием возможностей API-ключей нужна миграция Prisma.
- После установки зависимостей нужно сгенерировать клиент Prisma, чтобы был доступен
  `prisma.apiKey`.
- API-клиенты должны отправлять ключи в заголовке `x-api-key`.
- Секрет ключа показывается только один раз при создании.
- Обычное сохранение формы редактирования не продлевает срок действия; продление включается явно.
- Рекомендуется Node.js 22+, потому что текущая зависимость обработчика кеша требует эту версию.

## Качество

Релиз добавляет требования OpenSpec, покрытие Jest для конфигурации API-ключей, прав, схем,
действий, обработчиков маршрутов, вспомогательных функций управления и поведения UI, а также
Playwright-проверки для персональных ключей, ключей организаций и границ прав организации.
