# v0.0.2

v0.0.2 добавляет переиспользуемую основу локализации для основных сценариев шаблона. В релиз входят
`next-intl`, каталоги сообщений на английском и русском, локализованные метаданные и переведенные
тексты основного UI.

## Главное

- `next-intl` добавлен в стек шаблона.
- Добавлены каталоги сообщений на английском и русском для основных областей функций.
- Локализованы заголовки страниц, описания, метаданные Open Graph и метаданные Twitter.
- Основные тексты UI перенесены в пространства имен переводов вместо строк, зашитых в код.
- Добавлены общие вспомогательные функции для переводов на сервере и клиенте.
- Примеры окружения обновлены для настройки локали и аналитики.

## Что изменилось для пользователей шаблона

В шаблоне появилась базовая двуязычная настройка с поддержкой `en` и `ru`. Метаданные маршрутов,
формы, навигация, страницы авторизации, настройки аккаунта, экраны управления рабочими
пространствами и общие тексты UI читают значения из каталогов сообщений.

Локаль по умолчанию управляется через `PUBLIC_DEFAULT_LOCALE`.

## Заметки по обновлению

- `next-intl` входит в базовый стек.
- Файлы переводов находятся в `src/messages/`.
- Локаль по умолчанию задается через `PUBLIC_DEFAULT_LOCALE`.
- Yandex Metrika теперь ожидает `NEXT_PUBLIC_YM_COUNTER_ID`.

## Качество

Релиз добавляет тесты для настройки i18n-запросов, вспомогательных функций перевода страниц,
локализованных метаданных, переведенных компонентов среды выполнения, локализованных форм,
валидационных текстов из схем, доступа к окружению и поведения глобальной ошибки.
