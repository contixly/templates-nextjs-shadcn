# v0.0.7

v0.0.7 добавляет optional Redis/Valkey-backed caching для Cache Components и ISR, улучшает
workspace loading states и скрывает social login providers, пока они не настроены полностью.

## Главное

- Добавлены optional Redis/Valkey cache handlers для Next.js Cache Components и ISR.
- Добавлены shared remote-cache settings, connection URL handling и key prefixes.
- Добавлен local memory fallback, когда remote caching отключен.
- Добавлены workspace и dashboard Suspense boundaries с route-shaped skeleton states.
- Добавлены sidebar navigation primitives для dashboard access, workspace creation, secondary links
  и user account menus.
- Better Auth получает и показывает только полностью настроенные social providers.

## Что изменилось для пользователей шаблона

Single-instance local development использует local cache fallback. Multi-instance deployments могут
включить `REMOTE_CACHING_ENABLED=true` с `REDIS_URL` или `VALKEY_URL` и уникальным
`REMOTE_CACHING_PREFIX`.

Workspace pages получили более стабильные loading states для dashboard, users, teams, invitations,
roles и settings navigation. Login и account connection screens больше не показывают OAuth
providers с неполным набором environment variables.

## Upgrade notes

- Настройте `REDIS_URL` или `VALKEY_URL` до включения remote caching.
- Используйте уникальный `REMOTE_CACHING_PREFIX` для каждого app или environment при общем
  Redis/Valkey service.
- OAuth provider credentials должны быть полными; частично настроенные providers остаются скрытыми.
- Built-in Next.js in-memory cache отключен в пользу configured cache handlers.

## Качество

Релиз добавляет тесты для cache configuration, Redis/Valkey settings parsing, provider filtering,
login provider visibility, connection unlink eligibility, navigation behavior, workspace switchers,
streaming boundaries, settings shell behavior и document header rendering.
