# v0.0.5

v0.0.5 добавляет workspace-level allowed email-domain policies для приглашений и member
management. Релиз также показывает существующих участников вне активной политики и расширяет
shadcn/ui component foundation.

## Главное

- Добавлены workspace settings для allowed email domains в organization metadata.
- Allowed-domain restrictions применяются при создании workspace invitations.
- Invitation acceptance повторно проверяет текущую domain policy workspace.
- Direct member add показывает warning и требует explicit override для out-of-policy users.
- Users page показывает warnings и badges для существующих участников вне active policy.
- Добавлены domain normalization, validation, exact-domain matching и localized error copy.
- Добавлены дополнительные shadcn/ui components и custom loading button helper.

## Что изменилось для пользователей шаблона

Workspace admins могут настроить allowed email domains в workspace settings. Домены можно вводить по
одному на строку или через запятую, они нормализуются перед сохранением, а пустой список отключает
restriction.

Политика влияет на новые invitations и invitation acceptance. Existing members не удаляются при
изменении политики: member lists показывают warnings и badges, чтобы admins могли принять решение.
Добавить существующего out-of-policy user напрямую все еще можно, но только после explicit
confirmation.

## Upgrade notes

- Prisma migration для allowed-domain restrictions не требуется; policy хранится в organization
  metadata.
- Domain matching точный: `example.com` не разрешает автоматически `team.example.com`.
- Existing workspace members остаются участниками после изменения policy.
- Existing pending invitations могут стать restricted, если policy изменилась после создания.
- Custom behavior для generated shadcn components должен жить в `src/components/ui/custom/`.

## Качество

Релиз добавляет тесты для domain normalization, validation, metadata handling, invitation creation,
invitation acceptance, direct member-add warnings, workspace settings, users-page warnings,
domain-restricted invitation states и stable loading-button behavior.
