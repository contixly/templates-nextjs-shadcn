# v0.0.5

v0.0.5 добавляет политики разрешенных email-доменов рабочего пространства для приглашений и
управления участниками. Релиз также показывает существующих участников вне активной политики и
расширяет основу компонентов shadcn/ui.

## Главное

- В настройки рабочего пространства добавлены списки разрешенных доменов email в метаданных организации.
- Ограничения разрешенных доменов применяются при создании приглашений в рабочее пространство.
- Принятие приглашения повторно проверяет текущую доменную политику рабочего пространства.
- Прямое добавление участника показывает предупреждение и требует явного подтверждения для пользователей вне политики.
- Страница участников показывает предупреждения и бейджи для существующих участников вне активной политики.
- Добавлены нормализация доменов, валидация, точное сопоставление домена и локализованные тексты ошибок.
- Добавлены дополнительные компоненты shadcn/ui и собственная вспомогательная кнопка загрузки.

## Что изменилось для пользователей шаблона

Администраторы рабочего пространства могут настроить список разрешенных доменов email в настройках
рабочего пространства. Домены можно вводить по одному на строку или через запятую, они
нормализуются перед сохранением, а пустой список отключает ограничение.

Политика влияет на новые приглашения и принятие приглашений. Существующие участники не удаляются
при изменении политики: списки участников показывают предупреждения и бейджи, чтобы администраторы
могли принять решение. Добавить существующего пользователя вне политики напрямую все еще можно, но
только после явного подтверждения.

## Заметки по обновлению

- Миграция Prisma для ограничений разрешенных доменов не требуется; политика хранится в метаданных организации.
- Сопоставление доменов точное: `example.com` не разрешает автоматически `team.example.com`.
- Существующие участники рабочего пространства остаются участниками после изменения политики.
- Существующие ожидающие приглашения могут стать ограниченными, если политика изменилась после создания.
- Собственное поведение для сгенерированных компонентов shadcn должно жить в `src/components/ui/custom/`.

## Качество

Релиз добавляет тесты для нормализации доменов, валидации, обработки метаданных, создания
приглашений, принятия приглашений, предупреждений при прямом добавлении участника, настроек
рабочего пространства, предупреждений на странице участников, состояний приглашений с доменным
ограничением и стабильного поведения кнопки загрузки.
