# v0.0.11

v0.0.11 превращает `/docs` в полноценный публичный раздел документации шаблона. В версии появились
страницы Markdown и MDX на русском и английском языках, поиск по документации, страницы истории
версий и еженедельных изменений, а также требования OpenSpec для механизма документации.

<Callout title="Фокус версии" variant="info">
  Эта версия помогает быстрее находить нужную информацию. Пользователи шаблона могут читать
  документацию по продуктовым сценариям, API, рабочим пространствам, разработке, версиям и
  еженедельным изменениям в одном публичном разделе.
</Callout>

## 🚀 Новые возможности

- Добавлен публичный раздел `/docs` с общей оболочкой, боковым меню, хлебными крошками, действиями
  страницы, оглавлением, переключением темы, каноническими маршрутами и изображениями для ссылок
  [#16].
- Добавлен выбор страниц Markdown и MDX по языку интерфейса: русские и английские варианты сохраняют
  стабильные канонические URL, а при отсутствии нужного языка используется предсказуемый запасной
  вариант [#16].
- Опубликована двуязычная библиотека документации для управления аккаунтом, API-доступа, настроек
  приложения, рабочих пространств, материалов для разработчиков, истории версий и еженедельных
  изменений [#16].
- Добавлен поиск `/api/v1/documents-system/search`: он ищет страницы и заголовки с учетом выбранного
  языка, коротких запросов, раскладки клавиатуры, ограниченного числа опечаток и понятного пустого
  результата [#16].
- Добавлены компоненты MDX для выделенных блоков, вкладок, сеток ссылок, копирования кода, якорей
  заголовков, локальных изображений и безопасных внутренних ссылок [#16].

## 🐛 Исправления

- Строки внутри блоков кода с тильдами больше не создают ложные заголовки, якоря, пункты оглавления
  и результаты поиска [#16].
- Проверки внутренних ссылок, метаданных, бокового меню и оглавления стали строже, поэтому неверные
  цели публичной документации находятся раньше и отображаются предсказуемее [#16].
- Поиск по документации лучше обрабатывает короткие и составные запросы, когда точной фразы нет
  [#16].
- Для отсутствующих или скрытых страниц документации добавлена обработка "не найдено", при этом
  публичная документация остается доступной без авторизации [#16].

## ⚙️ Производительность и внутренние улучшения

- Добавлена возможность OpenSpec `documents-system`: маршруты, реестр страниц, языковые варианты,
  правила видимости, MDX-отрисовка, метаданные, ссылки, поиск, изображения, кеширование и режимы
  выполнения описаны как требования [#16].
- Добавлены целевые проверки Jest для разбора языка страниц, поведения реестра, метаданных страницы,
  бокового меню, оглавления, отслеживания прокрутки и проверки содержимого [#16].
- Подключены `@next/mdx`, пакеты MDX React, `glob`, `gray-matter`, плагины remark и `sharp` для
  локальных Markdown/MDX-страниц и изображений документации [#16].
- Обновлены инструкции для агентов и OpenSpec, чтобы следующая работа с документацией и требованиями
  опиралась на новую структуру публичного раздела [#16].
- Добавлены еженедельные заметки за 6-13 июля 2026 года в публичную историю документации.
- Метаданные пакета обновлены до `0.0.11` в `package.json` и `package-lock.json`.
