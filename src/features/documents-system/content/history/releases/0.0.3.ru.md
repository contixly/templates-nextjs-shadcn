# v0.0.3

v0.0.3 заменяет раннюю основу рабочих пространств на организации Better Auth. Релиз добавляет
маршрутизацию в рамках организации, настройки рабочего пространства, приглашения, управление
участниками, права с учетом ролей и онбординг для пользователя без рабочего пространства.

## Главное

- Добавлена поддержка организаций Better Auth для совместной работы в рабочих пространствах.
- Добавлены маршруты `/w/:organizationKey/...`, ограниченные конкретной организацией.
- `/dashboard` получил выбор активной организации и запасной вариант, если активная организация недоступна.
- Добавлены страницы настроек рабочего пространства для деталей, участников, приглашений, команд и ролей.
- Добавлены создание приглашений, список, копирование ссылки, принятие, отклонение и точки входа для ожидающих приглашений.
- Добавлены каталог участников, прямое добавление участника и управление ролями `owner`, `admin` и `member`.
- Добавлен онбординг для пользователей без доступных рабочих пространств.

## Что изменилось для пользователей шаблона

URL рабочих пространств теперь предпочитают `/w/:organizationKey/...`, где ключ может быть slug
организации или ее id. Переключатели рабочих пространств меняют контекст активной организации
только через явные действия пользователя и по возможности сохраняют соответствующие базовые
маршруты рабочего пространства.

Пользователи могут создавать и обновлять рабочие пространства, управлять участниками и ролями при
наличии прав, создавать приглашения и просматривать личные ожидающие приглашения в аккаунте или на
экране приветствия.

## Заметки по обновлению

- Рабочие пространства теперь работают на организациях Better Auth вместо отдельной модели рабочего пространства.
- Существующие развертывания должны выполнить новые миграции Prisma перед использованием сценариев организаций и приглашений.
- Принятие приглашения требует, чтобы подтвержденный основной email приглашенного пользователя совпадал с приглашением.
- Права рабочего пространства следуют проверкам прав организаций Better Auth и встроенным правилам ролей.

## Качество

Релиз добавляет тесты для защиты маршрутов организации, генерации slug, тегов кеша репозиториев,
создания, обновления и удаления рабочих пространств, управления участниками, назначения ролей,
приглашений, страниц настроек, защиты онбординга, переключателей в боковом меню, приглашений
аккаунта и потребителей локализованных метаданных.
