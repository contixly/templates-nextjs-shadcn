# v0.0.8

v0.0.8 добавляет local automation auth и Playwright E2E infrastructure. Релиз делает browser
automation удобной для локальной разработки и при этом оставляет automation flow явно отключенным в
production.

## Главное

- Добавлен opt-in local automation auth для Playwright, browser-use и agent-driven browser checks.
- Добавлен `/api/local-auth/scenario` для создания signed-in automation users и cleanup.
- Добавлена gated automation panel на странице входа для локального создания test user.
- Добавлены Playwright scripts, config, Chromium smoke coverage, reports, traces, screenshots и
  videos.
- Добавлены helpers для route warming и first-party error detection в E2E runs.
- Better Auth organization hooks теперь проверяют workspace domain и team invitation policy.

## Что изменилось для пользователей шаблона

Local automation users используют namespace `local-agent+...@local-agent.test` и доступны только при
`LOCAL_AUTOMATION_AUTH_ENABLED=true` вне production. Scenario route создает настоящий Better Auth
session cookie, поэтому browser tests проверяют protected pages так же, как signed-in user.

У Playwright появился стандартный локальный workflow: один раз установить Chromium, запустить
`npm run e2e` и позволить suite поднять development server с нужным automation environment.

## Upgrade notes

- Prisma migration для этого релиза не требуется.
- `LOCAL_AUTOMATION_AUTH_ENABLED` нельзя включать в production.
- `npm run e2e:install` нужно выполнить один раз на машинах, где еще нет Playwright browser.
- `npm run e2e` поднимает development server, если не указан `PLAYWRIGHT_START_SERVER=false`.
- Default Playwright base URL - `http://127.0.0.1:3127`.
- Playwright artifacts игнорируются в `playwright-report`, `test-results/playwright` и
  `blob-report`.

## Качество

Релиз добавляет тесты для local automation gating, credential generation, scenario create/cleanup,
login panel behavior, organization invitation policy hooks, route config behavior и public UI smoke
test для home-to-login flow.
