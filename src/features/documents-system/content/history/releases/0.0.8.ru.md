# v0.0.8

v0.0.8 добавляет локальную авторизацию для автоматизации и инфраструктуру Playwright E2E. Релиз
делает браузерную автоматизацию удобной для локальной разработки и при этом оставляет сценарий
автоматизации явно отключенным в боевой среде.

## Главное

- Добавлена включаемая явно локальная авторизация для Playwright, `browser-use` и браузерных проверок агентов.
- Добавлен `/api/local-auth/scenario` для создания вошедших пользователей автоматизации и очистки данных.
- На странице входа добавлена панель автоматизации с защитой для локального создания тестового пользователя.
- Добавлены скрипты Playwright, конфигурация, быстрые проверки Chromium, отчеты, трассировки, скриншоты и видео.
- Добавлены вспомогательные функции для прогрева маршрутов и обнаружения ошибок собственного приложения в E2E-запусках.
- Хуки организаций Better Auth теперь проверяют домен рабочего пространства и политику приглашений в команды.

## Что изменилось для пользователей шаблона

Пользователи локальной автоматизации используют пространство имен
`local-agent+...@local-agent.test` и доступны только при `LOCAL_AUTOMATION_AUTH_ENABLED=true` вне
боевой среды. Маршрут сценария создает настоящий cookie сессии Better Auth, поэтому браузерные
тесты проверяют защищенные страницы так же, как вошедший пользователь.

У Playwright появился стандартный локальный процесс: один раз установить Chromium, запустить
`npm run e2e` и позволить набору тестов поднять сервер разработки с нужным окружением автоматизации.

## Заметки по обновлению

- Миграция Prisma для этого релиза не требуется.
- `LOCAL_AUTOMATION_AUTH_ENABLED` нельзя включать в боевой среде.
- `npm run e2e:install` нужно выполнить один раз на машинах, где еще нет браузера Playwright.
- `npm run e2e` поднимает сервер разработки, если не указан `PLAYWRIGHT_START_SERVER=false`.
- Базовый URL Playwright по умолчанию - `http://127.0.0.1:3127`.
- Артефакты Playwright игнорируются в `playwright-report`, `test-results/playwright` и `blob-report`.

## Качество

Релиз добавляет тесты для включения локальной автоматизации, генерации учетных данных, создания и
очистки сценария, поведения панели входа, хуков политики приглашений организации, поведения
конфигурации маршрутов и быструю проверку публичного UI для пути от главной страницы до входа.
