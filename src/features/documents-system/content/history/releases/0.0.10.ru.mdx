# v0.0.10

v0.0.10 — релиз Next.js шаблона про E2E-покрытие и надежность automation-сценариев. Он расширяет
OpenSpec-backed Playwright проверки для аккаунта, workspace, приглашений, API-ключей, `/api/v1`,
security headers и local automation auth.

<Callout title="Фокус релиза" variant="info">
  Этот релиз не меняет продуктовую модель. Он делает существующие сценарии шаблона проще для
  проверки, безопаснее для локальной автоматизации и стабильнее при запуске тестов против уже
  поднятого dev-сервера.
</Callout>

## Главное

- Расширено Playwright-покрытие для account settings, account sessions, workspace management,
  workspace invitations, workspace teams, workspace users, workspace routing, workspace fallbacks,
  API key management, `/api/v1`, local automation auth и security headers.
- Добавлены общие E2E helpers для workspaces, invitations, browser contexts, API keys, local auth,
  route constants и cold-route retry handling.
- Playwright запускается последовательно по умолчанию, чтобы уменьшить конфликты состояния между
  сценариями.
- Добавлена явная настройка `AUTH_DISABLE_SESSION_COOKIE_CACHE` для local automation и
  reused-server Playwright запусков.
- API key configuration constants вынесены в одно место с сохранением совместимых re-exports.
- Workspace creation и settings navigation стали устойчивее: ключевые workspace paths
  revalidate-ятся после создания, а изменения slug или route key ведут на canonical settings route.

## Upgrade notes

- Prisma migration для этого релиза не требуется.
- Package metadata теперь показывает `0.0.10`; dependency upgrades в релиз не входят.
- Playwright запускается с `workers: 1` по умолчанию.
- Стандартный `npm run e2e` поднимает собственный dev-server и выставляет local automation и
  session-cache окружение, которое ожидает suite.
- Reused-server запуск с `PLAYWRIGHT_START_SERVER=false` требует
  `AUTH_DISABLE_SESSION_COOKIE_CACHE=true` и в уже поднятом приложении, и в Playwright command.
- `LOCAL_AUTOMATION_AUTH_ENABLED` остается только локальной настройкой и не должен включаться в
  production.
- Изменения workspace slug или route key после сохранения ведут на canonical settings routes.

## Качество

В релиз добавлены и расширены OpenSpec-backed Playwright specs для account, API key, local
automation, settings, security и workspace capabilities. Также добавлены infrastructure tests для
API key E2E helpers, Playwright global setup, proxy session-cache behavior, централизованной API key
configuration, account actions, local automation cleanup и workspace settings.
