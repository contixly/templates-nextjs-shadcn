# v0.0.10

v0.0.10 — релиз Next.js шаблона про E2E-покрытие и надежность сценариев автоматизации. Он расширяет
Playwright-проверки на базе OpenSpec для аккаунта, рабочего пространства, приглашений, API-ключей,
`/api/v1`, заголовков безопасности и локальной авторизации для автоматизации.

<Callout title="Фокус релиза" variant="info">
  Этот релиз не меняет продуктовую модель. Он делает существующие сценарии шаблона проще для
  проверки, безопаснее для локальной автоматизации и стабильнее при запуске тестов против уже
  поднятого сервера разработки.
</Callout>

## Главное

- Расширено Playwright-покрытие для настроек аккаунта, сессий аккаунта, управления рабочим
  пространством, приглашений, команд, участников, маршрутизации, запасных сценариев рабочего
  пространства, управления API-ключами, `/api/v1`, локальной авторизации для автоматизации и
  заголовков безопасности.
- Добавлены общие вспомогательные функции E2E для рабочих пространств, приглашений, браузерных
  контекстов, API-ключей, локальной авторизации, констант маршрутов и повторов холодных маршрутов.
- Playwright запускается последовательно по умолчанию, чтобы уменьшить конфликты состояния между
  сценариями.
- Добавлена явная настройка `AUTH_DISABLE_SESSION_COOKIE_CACHE` для локальной автоматизации и
  запусков Playwright против уже поднятого сервера.
- Константы конфигурации API-ключей вынесены в одно место с сохранением совместимых повторных экспортов.
- Создание рабочего пространства и навигация настроек стали устойчивее: ключевые пути рабочего
  пространства обновляются после создания, а изменения slug или ключа маршрута ведут на
  канонический маршрут настроек.

## Заметки по обновлению

- Миграция Prisma для этого релиза не требуется.
- Метаданные пакета теперь показывают `0.0.10`; обновления зависимостей в релиз не входят.
- Playwright запускается с `workers: 1` по умолчанию.
- Стандартный `npm run e2e` поднимает собственный сервер разработки и выставляет окружение локальной
  автоматизации и кеша сессий, которое ожидает набор тестов.
- Запуск против уже поднятого сервера с `PLAYWRIGHT_START_SERVER=false` требует
  `AUTH_DISABLE_SESSION_COOKIE_CACHE=true` и в уже поднятом приложении, и в команде Playwright.
- `LOCAL_AUTOMATION_AUTH_ENABLED` остается только локальной настройкой и не должен включаться в
  боевой среде.
- Изменения slug рабочего пространства или ключа маршрута после сохранения ведут на канонические маршруты настроек.

## Качество

В релиз добавлены и расширены спецификации Playwright на базе OpenSpec для возможностей аккаунта,
API-ключей, локальной автоматизации, настроек, безопасности и рабочего пространства. Также
добавлены инфраструктурные тесты для E2E-вспомогательных функций API-ключей, глобальной настройки
Playwright, поведения кеша сессий в прокси, централизованной конфигурации API-ключей, действий
аккаунта, очистки локальной автоматизации и настроек рабочего пространства.
