# v0.0.4

v0.0.4 усиливает настройки безопасности среды выполнения, перенаправления авторизации, границы
защищенных действий, управление сессиями аккаунта и приватность приглашений в рабочее пространство.

## Главное

- Добавлены общие заголовки безопасности приложения для встраивания в iframe, типа контента, referrer, permissions, CSP и HSTS в боевой среде.
- Временный базовый URL заменен на нормализованную конфигурацию `NEXT_PUBLIC_APP_BASE_URL` и `BETTER_AUTH_URL`.
- Перенаправления авторизации защищены от внешних адресов и переходов через `..`.
- Защищенные серверные действия больше не доверяют переданным заголовкам с идентификатором пользователя.
- Сырые токены сессий Better Auth удалены из данных сессии, доступных клиентскому UI.
- Метаданные приглашения в рабочее пространство скрываются, если вошедший пользователь не является адресатом приглашения.
- Добавлена защита базы данных от дублирующихся ожидающих приглашений.
- Уникальность slug рабочего пространства проверяется среди всех организаций.

## Что изменилось для пользователей шаблона

Боевые развертывания теперь быстро падают при запуске, если не задан публичный базовый URL
приложения. Цели перенаправления авторизации ограничены безопасными внутренними путями приложения.
Страницы безопасности аккаунта показывают безопасные метаданные сессии без сырых токенов сессий.

Страницы приглашений в рабочее пространство защищают приватность адресата, а принятие приглашения
устанавливает принятое рабочее пространство как активную организацию.

## Заметки по обновлению

- Замените `PUBLIC_BASE_URL` на `NEXT_PUBLIC_APP_BASE_URL` во всех окружениях.
- Боевые развертывания должны задавать `NEXT_PUBLIC_APP_BASE_URL`.
- Нужна миграция Prisma для индекса защиты от дублирующихся приглашений.
- Внешние изображения больше не разрешены с любых HTTPS-хостов; настройте `images.remotePatterns` для нужных доменов.
- Изменения названия рабочего пространства больше не пересоздают slug автоматически.

## Качество

Релиз добавляет тесты для очистки перенаправлений, обработки токенов сессий, защиты от отзыва
текущей сессии, приватности при несовпадении адресата приглашения, принятия приглашений,
обновления активной организации, глобальных конфликтов slug и формы данных списка сессий.
