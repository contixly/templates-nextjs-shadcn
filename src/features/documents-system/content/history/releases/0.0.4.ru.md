# v0.0.4

v0.0.4 усиливает runtime security defaults, auth redirects, protected action boundaries, account
session management и workspace invitation privacy.

## Главное

- Добавлены application-wide security headers для framing, content type, referrer, permissions, CSP
  и production HSTS.
- Placeholder base URL заменен на normalized `NEXT_PUBLIC_APP_BASE_URL` и `BETTER_AUTH_URL`
  configuration.
- Auth redirects защищены от external и traversal-based targets.
- Protected server actions больше не доверяют forwarded user-id headers.
- Raw Better Auth session tokens удалены из client-facing session data.
- Workspace invitation metadata скрывается, если signed-in user не является invited recipient.
- Добавлена database protection от duplicate pending invitations.
- Workspace slug uniqueness проверяется среди всех organizations.

## Что изменилось для пользователей шаблона

Production deployments теперь fail fast, если не задан public app base URL. Auth redirect targets
ограничены safe in-app paths. Account security pages показывают безопасную session metadata без raw
bearer-like session tokens.

Workspace invitation pages защищают privacy recipient, а acceptance устанавливает accepted
workspace как active organization.

## Upgrade notes

- Замените `PUBLIC_BASE_URL` на `NEXT_PUBLIC_APP_BASE_URL` во всех окружениях.
- Production deployments должны задавать `NEXT_PUBLIC_APP_BASE_URL`.
- Нужна Prisma migration для invitation duplicate-protection index.
- Remote images больше не разрешены с любых HTTPS hosts; настройте `images.remotePatterns` для
  нужных доменов.
- Workspace name changes больше не regenerates slugs автоматически.

## Качество

Релиз добавляет тесты для redirect sanitization, session-token handling, current-session revoke
protection, invitation recipient mismatch privacy, accepting invitations, active organization
updates, global slug collisions и session-list data shape.
