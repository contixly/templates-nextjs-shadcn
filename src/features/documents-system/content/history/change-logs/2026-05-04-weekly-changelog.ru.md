# Еженедельное обновление - 27 апреля-4 мая 2026

## ✨ Новые возможности

- **Команды рабочего пространства**: временный раздел настроек команд заменен на поддержку Better Auth Teams: создание, переименование, удаление команд, добавление и удаление участников, а также режим только для чтения для обычных участников.

- **Приглашения в команду**: приглашение теперь может быть привязано к конкретной команде рабочего пространства. Таблицы приглашений, карточки ожидающих приглашений и страницы решения показывают целевую команду, а принятое приглашение добавляет пользователя в нее.

- **Распределенное кэширование**: добавлены опциональные обработчики кеша Redis/Valkey для Cache Components и ISR, общие настройки окружения и локальный запасной вариант.

- **Локальная авторизация для автоматизации и Playwright E2E**: добавлен локальный сценарий входа для автоматизации, `/api/local-auth/scenario`, скрипты Playwright, быстрые проверки Chromium, прогрев маршрутов и проверка ошибок собственного приложения.

## 🔧 Улучшения

- **Состояния загрузки рабочего пространства**: панель и настройки рабочего пространства теперь используют локальные границы Suspense и заглушки загрузки вместо общего запасного экрана загрузки маршрута.

- **Настроенные OAuth-провайдеры**: вход, привязка аккаунтов и навигация показывают только OAuth-провайдеры, для которых заданы обязательные переменные окружения.

- **Полировка навигации**: добавлены более насыщенная боковая навигация, быстрый доступ к созданию рабочего пространства, дополнительные ссылки, меню пользователя и явное закрытие мобильного бокового меню.

- **Обратная связь в формах**: UI загрузки и валидации переведен на общие компоненты кнопок, сообщений полей и ошибок формы, чтобы формы меньше прыгали при отправке и ошибках.

## 🐛 Исправления

- **Проверки политик приглашений**: хуки организаций Better Auth теперь применяют политики email-доменов и целевых команд даже для прямых вызовов плагина, обходящих действия функции.

- **Безопасность подключенных аккаунтов**: пользователь не может отвязать последний настроенный OAuth-провайдер, который еще дает доступ к аккаунту.

## 📝 Документация

- Добавлены заметки к релизам по командам, распределенному кэшированию и локальной автоматизации. Обновлены E2E-документация, настройка окружения, README, спецификации OpenSpec и инструкции для агентов по браузерной автоматизации.
