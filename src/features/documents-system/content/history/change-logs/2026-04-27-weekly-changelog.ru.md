# Еженедельное обновление - 20-27 апреля 2026

## ✨ Новые возможности

- **Рабочие пространства на организациях**: workspaces теперь работают поверх Better Auth organizations, но в интерфейсе сохраняют пользовательский термин "Workspace". Добавлены organization-scoped маршруты `/w/:organizationKey`.

- **Настройки и совместная работа в workspace**: появились страницы настроек workspace, пользователей, приглашений, команд и ролей, а также каталог участников, управление ролями, создание приглашений, review приглашений и onboarding для пользователя без workspace.

- **Политики разрешённых email-доменов**: администратор workspace может задать разрешённые домены, применять их к приглашениям и видеть предупреждения, если текущие участники находятся вне активной политики.

## 🔧 Улучшения

- **Безопасные настройки по умолчанию**: добавлены browser security headers, production HSTS, строгая политика внешних изображений, безопасная обработка auth redirect и нормализованная настройка публичного URL приложения.

- **Приватность приглашений и участников**: детали приглашения скрываются, если текущий пользователь не является адресатом, повторные pending-приглашения блокируются, а принятие приглашения переключает активный workspace.

- **Защита идентичности workspace**: slug проверяется среди всех организаций, а переименование workspace больше не меняет slug без явной отправки нового значения.

## 🐛 Исправления

- **Граница protected actions**: защищённые server actions теперь используют проверенную Better Auth session и не доверяют forwarded user-id headers.

- **Безопасность списка сессий**: список сессий аккаунта больше не отдаёт в клиентский UI raw Better Auth session tokens.

## 📝 Документация

- Добавлены release notes по organization-backed workspaces, security hardening и domain restrictions. Обновлены README, setup notes и OpenSpec requirements для workspace management и invitations.
