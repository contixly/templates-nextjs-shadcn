# Оболочка настроек

Общая оболочка настроек делает настройки аккаунта и рабочего пространства предсказуемыми.
Пользователь должен видеть единую структуру страницы и при редактировании профиля, и при
управлении участниками рабочего пространства.

## Структура страницы

Каждая страница настроек начинается с контекстного вступления. Ниже контент группируется в
разделы:

- формы группируются по области решения;
- таблицы и списки группируются в разделы, удобные для просмотра;
- пустые состояния остаются внутри своего раздела.

## Режимы ширины

Плотные таблицы, например участники или команды, могут использовать более широкий макет.
Сфокусированные формы, например профиль или детали рабочего пространства, остаются в удобной для
чтения ширине.

## Поведение тем

Страницы настроек остаются читаемыми в светлой и темной темах. Опасные области визуально
отличаются, но не доминируют на странице.

## Правила расширения

Новые разделы настроек должны использовать ту же оболочку и не создавать собственное оформление
страницы. Добавляйте понятное вступление, группируйте контент по задаче и показывайте элементы
управления только тогда, когда текущий пользователь имеет право на действие.
