# Локализация

Шаблон включает каталоги сообщений на английском и русском языках и локаль по умолчанию, выбранную
через `PUBLIC_DEFAULT_LOCALE`. Та же настройка управляет сообщениями приложения и контентом
документации.

## Поддерживаемые локали

Поддерживаемые локали - `en` и `ru`. Если `PUBLIC_DEFAULT_LOCALE` пустая или неподдерживаемая,
шаблон возвращается к `en`.

## UI-сообщения

Пользовательские тексты интерфейса находятся в `src/messages`. Новая UI-часть функции должна
добавлять сообщения в обеих поддерживаемых локалях и использовать существующую структуру
пространств имен.

## Метаданные страниц

Страницы используют вспомогательные функции метаданных маршрутов, чтобы заголовки, описания, данные
Open Graph и другие метаданные брались из локализованных сообщений.

## Контент документации

Файлы документации используют суффиксы локали: например `.en.md` и `.ru.md`. URL остается
каноническим: `/docs/workspace/settings` - один маршрут для обоих языков.

Если документ существует только в одной локали, система документации может использовать его как
запасной контент и показать языковой маркер в UI.

## Заметка про Cache Components

Шаблон работает с включенным Next.js Cache Components. Поддерживаемая модель развертывания - одна
настроенная локаль по умолчанию на экземпляр приложения или домен.
