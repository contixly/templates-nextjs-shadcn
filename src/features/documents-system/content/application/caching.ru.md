# Кеширование

Шаблон использует Next.js Cache Components и теги кеша на уровне репозиториев. Он может работать с
локальным запасным кешем или распределенным кешем на Redis/Valkey.

## Локальное значение по умолчанию

По умолчанию `REMOTE_CACHING_ENABLED=false`. Это упрощает локальную разработку и использует
локальный запасной вариант из пользовательских обработчиков кеша.

## Распределенный кеш

Включайте распределенное хранилище кеша, когда несколько экземпляров приложения должны делить Cache
Components и записи ISR:

| Переменная | Назначение |
| ---------- | ---------- |
| `REMOTE_CACHING_ENABLED` | Включает удаленный режим кеша при значении `true`. |
| `REDIS_URL` или `VALKEY_URL` | URL подключения к сервису кеша. |
| `REDIS_PASSWORD` | Опциональный пароль, если он не включен в URL. |
| `REMOTE_CACHING_PREFIX` | Префикс, который изолирует записи кеша по приложению или окружению. |

Используйте уникальный префикс, если один сервис Redis или Valkey делят несколько окружений.

## Инвалидация кеша функций

Чтение из репозиториев использует теги кеша. Мутации обновляют затронутые теги и пути, чтобы UI
показывал свежие данные рабочих пространств, аккаунтов, команд, приглашений и API-ключей.
