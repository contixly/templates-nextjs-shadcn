# Оболочка приложения

Шаблон поставляется с публичными маршрутами, защищенными маршрутами, общей оболочкой приложения и
стартовой панелью. Эти поверхности предполагают замену или расширение продуктом, который создается
на базе шаблона.

## Публичные маршруты

Публичные маршруты включают главную страницу, страницы входа и ошибки, документацию, конечные
точки авторизации, проверки состояния и `/api/v1`. Публичность не означает доступ к данным без
авторизации: API v1 все равно требует аутентификацию по API-ключу.

## Защищенные маршруты

Защищенные страницы приложения находятся за проверками сессии Better Auth. Граница маршрута
применяется до рендера защищенного контента.

Глобальные защищенные маршруты:

- `/dashboard`;
- `/welcome`;
- `/workspaces`;
- `/user/...`;
- `/invite/:invitationId`;
- `/w/:organizationKey/...`.

## Стартовая панель

Стартовая панель - демонстрационная страница шаблона. Она показывает защищенный макет, контекст
маршрута рабочего пространства, заглушку загрузки и общие UI-примитивы. При разработке сервиса
замените демонстрационный контент на обзорные данные продукта.

## Точки расширения

Используйте существующие функциональные срезы как основной паттерн для добавления продуктовых
возможностей:

- описывайте маршруты в файле маршрутов функции;
- оставляйте страницы и макеты тонкими в `src/app`;
- держите бизнес-логику в `src/features`;
- используйте общие UI-примитивы из `src/components`.
