# Профиль и подключения

Страница профиля отделяет данные пользователя от подключений провайдеров. Так базовые данные
аккаунта легко проверить, а продуктовая команда может включать только нужных OAuth-провайдеров.

## Обновить отображаемое имя

1. Откройте `/user/profile`.
2. Измените отображаемое имя.
3. Сохраните форму.

Действие проверяет новое значение и отклоняет пустые или неизмененные отправки. Email показывается
как идентификатор аккаунта и не редактируется через эту форму.

## Управлять подключенными провайдерами

Откройте `/user/connections`, чтобы увидеть доступных OAuth-провайдеров. В UI показываются только
провайдеры с полным набором переменных окружения.

Пользователь может подключить доступного провайдера, который еще не связан с аккаунтом. Отключение
разрешено только тогда, когда аккаунт не остается без безопасного способа входа.

## Конфигурация провайдеров

Шаблон поддерживает Google, GitHub, GitLab, VK и Yandex через Better Auth. Провайдер появляется на
страницах входа и подключений только после настройки обязательных переменных окружения.
