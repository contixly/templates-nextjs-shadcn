# Сессии и безопасность

Страница безопасности помогает пользователю смотреть активные сессии и отзывать доступ без вывода
сырых токенов сессии в браузер.

## Посмотреть сессии

Откройте `/user/security`, чтобы увидеть метаданные сессий: устройство, расположение и время
активности. Текущая сессия отделена от остальных.

Страница намеренно показывает только безопасные идентификаторы. Секретные токены сессий
определяются и обрабатываются на сервере.

## Отозвать одну сессию

Используйте действие в строке сессии, которая не является текущей. Сервер найдет токен выбранной
сессии и отзовет его.

Текущую сессию нельзя отозвать через действие для одной сессии. Это предотвращает сценарий, в
котором пользователь случайно отзывает страницу, которой сейчас пользуется.

## Отозвать все остальные сессии

Используйте массовое действие, чтобы выйти из всех других активных сессий и сохранить текущую.
Это полезно после проблем с паролем, провайдером или устройством.

## Ожидания безопасности

Защищенные действия аккаунта игнорируют поддельные клиентские данные пользователя и всегда
загружают аутентифицированную сессию на сервере. Продуктовые функции на базе шаблона должны
использовать тот же паттерн защищенных действий.
