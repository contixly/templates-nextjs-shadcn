# Участники и роли

Страница настроек участников показывает участников рабочего пространства и открывает элементы
управления с учетом роли.

## Встроенные роли

Шаблон стартует с тремя управляемыми ролями:

| Роль | Типичная ответственность |
| ---- | ------------------------ |
| Owner | Полное владение рабочим пространством, включая опасные действия с ним. |
| Admin | Управление рабочим пространством: участниками, командами, приглашениями и настройками. |
| Member | Доступ на чтение к безопасному контексту рабочего пространства и продуктовым поверхностям. |

Продуктовая команда может расширять поведение ролей, но видимое значение каждой роли должно
оставаться понятным в настройках рабочего пространства.

## Каталог участников

Все участники с доступом могут смотреть каталог пользователей. Текущий пользователь выделен отдельно, а
остальные участники показаны в таблице.

Если участники не возвращены, страница оставляет пустое состояние внутри раздела пользователей и не
теряет контекст настроек.

## Добавить существующего пользователя

Уполномоченные участники могут добавить существующего пользователя по id. Ограничения по доменам
проверяются до создания участия. Если пользователь вне активной политики, UI может потребовать
явное подтверждение перед продолжением.

## Изменить роли

Уполномоченные участники могут менять назначаемые роли из таблицы участников. Шаблон отклоняет
запрещенные, избыточные или неавторизованные изменения ролей.

Удаление участников намеренно не выведено в текущей стартовой поверхности.
