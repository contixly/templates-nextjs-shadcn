# Создание и переключение рабочего пространства

Рабочие пространства - это пользовательские пространства совместной работы в шаблоне. Каждое
рабочее пространство использует организацию Better Auth и имеет маршрут `/w/:organizationKey/...`.

## Создать рабочее пространство

1. Откройте страницу управления рабочими пространствами из оболочки приложения или состояния онбординга.
2. Нажмите **Создать рабочее пространство**.
3. Введите название рабочего пространства.
4. Сохраните форму.

Приложение создает базовую организацию и генерирует уникальный slug для URL рабочего пространства.
Новое рабочее пространство не создает команду по умолчанию; ноль явных команд считается нормальным
состоянием.

## Открыть рабочее пространство

Страницы рабочего пространства используют `/w/:organizationKey/...`, где `organizationKey` может
быть slug или id организации. UI-ссылки предпочитают slug, но ссылки на базе id остаются валидными
для совместимости.

При открытии маршрута рабочего пространства приложение проверяет доступ до рендера контента.
Недоступные маршруты показывают защищенное состояние и не раскрывают данные рабочего пространства.

## Переключить контекст рабочего пространства

Используйте переключатель рабочих пространств в оболочке приложения, чтобы перейти между
доступными рабочими пространствами. Для базовых маршрутов рабочего пространства переключатель
сохраняет эквивалентное назначение в выбранном рабочем пространстве. Для неизвестных или сложных
путей он переходит на панель выбранного рабочего пространства.

Глубокие ссылки используют рабочее пространство из URL. Они не переписывают контекст активного
рабочего пространства в сессии без явного выбора пользователя.

## Удалить рабочее пространство

Удаление рабочего пространства доступно только участникам с нужным правом удаления организации.
Шаблон защищает от небезопасного удаления, после которого пользователь остался бы без доступной
рабочей области.
