# Команды

Команды - это явные подгруппы внутри рабочего пространства. Они используют Better Auth Teams, но в
интерфейсе остаются функцией рабочего пространства.

## Модель команд

Новое рабочее пространство не создает команду по умолчанию. Организация рабочего пространства
остается контекстом для всех участников, а ноль явных команд считается валидным состоянием.

Названия команд уникальны внутри одного рабочего пространства. Участие в команде из другого
рабочего пространства отклоняется.

## Управлять командами

Уполномоченные участники могут:

- создать команду;
- переименовать команду;
- удалить команду, включая последнюю явную команду;
- добавить существующего участника рабочего пространства в команду;
- удалить участника команды.

Обычные участники могут смотреть команды без элементов управления.

## Приглашения в команду

Приглашение может опционально указывать целевую команду. После принятия приглашенный пользователь
вступает в рабочее пространство и выбранную команду. Приглашения без целевой команды остаются
только приглашениями в рабочее пространство.

## Нет управления активной командой

Шаблон не выводит переключение активной команды или изменение активной команды. Команды
управляются как подгруппы, а не как второй контекст маршрутизации.
