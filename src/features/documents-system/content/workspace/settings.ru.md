# Настройки рабочего пространства

Настройки рабочего пространства разделены на отдельные страницы. Так каждая управленческая задача
остается сфокусированной, а навигация показывает только те разделы, которые доступны текущему
участнику.

## Разделы настроек

| Раздел | Маршрут | Что содержит |
| ------ | ------- | ------------ |
| Рабочее пространство | `/w/:organizationKey/settings/workspace` | Название рабочего пространства, slug и список разрешенных доменов email. |
| Участники | `/w/:organizationKey/settings/users` | Сводка текущего пользователя, список участников, прямое добавление участников и управление ролями. |
| Приглашения | `/w/:organizationKey/settings/invitations` | Таблица приглашений, статусы, ссылки и модальное окно создания приглашения. |
| Команды | `/w/:organizationKey/settings/teams` | Явные команды рабочего пространства и управление участниками команд. |
| Роли | `/w/:organizationKey/settings/roles` | Запланированный раздел для будущего расширения ролей. |
| API-ключи | `/w/:organizationKey/settings/api-keys` | API-ключи рабочего пространства для `/api/v1`. |

Корневой маршрут настроек перенаправляет в первый доступный раздел после проверки доступа.

## Права

Обычные участники могут читать безопасный контекст настроек: например, списки участников и команды.
Элементы управления появляются только у участников, роль которых разрешает действие.

Разделы приглашений и API-ключей условные: пользователи без нужных прав не видят управляющие
действия, а иногда не видят и ссылку на раздел.

## Общая оболочка настроек

Настройки аккаунта и рабочего пространства используют общую визуальную оболочку: контекстное
вступление сверху, затем разделы для форм, списков и пустых состояний. Это сохраняет читаемость
плотных операционных страниц в светлой и темной темах.
