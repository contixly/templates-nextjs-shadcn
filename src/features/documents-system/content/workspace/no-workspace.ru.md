# Пользователь без рабочего пространства

У пользователя с активной сессией временно может не быть доступных рабочих пространств: он новый,
удален из организации или ждет приглашение. Шаблон обрабатывает это состояние через
переиспользуемый защитный механизм онбординга.

## Что делает защитный механизм

Защитный механизм не дает защищенному контенту рабочего пространства отрендериться без валидного
контекста организации. Вместо этого он направляет пользователя к действиям, которые могут вернуть
доступ:

- создать рабочее пространство;
- посмотреть ожидающие приглашения;
- открыть настройки аккаунта.

## Что остается доступным

Состояние без рабочего пространства не блокирует глобальные страницы управления. Пользователь все
еще может открыть настройки аккаунта, страницу управления рабочими пространствами и личный список
приглашений.

## Поведение панели

Глобальный маршрут `/dashboard` выбирает лучшее доступное рабочее пространство. Если активное
рабочее пространство невалидно, приложение переходит к стабильному доступному рабочему
пространству. Если рабочих пространств нет, пользователь попадает на экран приветствия и
первичной настройки.
