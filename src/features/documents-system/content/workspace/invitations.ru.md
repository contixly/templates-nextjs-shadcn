# Приглашения

Приглашения в рабочее пространство позволяют администратору пригласить пользователя по email и
опционально добавить его в команду после принятия.

## Создать приглашение

1. Откройте настройки рабочего пространства.
2. Перейдите в **Приглашения**.
3. Нажмите **Создать приглашение**.
4. Введите email получателя.
5. Выберите роль и, если нужно, целевую команду.
6. Скопируйте ссылку приглашения.

Шаблон отклоняет дублирующиеся, избыточные, ограниченные доменом и неавторизованные запросы
приглашений.

## Статусы приглашений

Статус приглашения стабильно выводится из сохраненного состояния и срока действия. Таблица
администратора показывает ожидающие, принятые, отклоненные, истекшие или иначе завершенные
приглашения в одном месте.

## Принять или отклонить приглашение

Приглашенный пользователь открывает ссылку приглашения после входа. Анонимный посетитель
возвращается на маршрут приглашения после входа.

Страница решения показывает пригласившего, рабочее пространство, роль, срок действия и целевую
команду, если она выбрана. Принятие требует, чтобы подтвержденный основной email пользователя
совпадал с приглашением и активной политикой доменов.

## Личный список приглашений

Пользователь может смотреть свои ожидающие приглашения на `/user/invitations`. Страница
приветствия переиспользует тот же блок ожидающих приглашений для пользователей без рабочего
пространства.
