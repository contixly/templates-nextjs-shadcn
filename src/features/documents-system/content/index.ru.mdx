# Документация шаблона

Эта документация объясняет, что уже есть в шаблоне приложения Next.js и как использовать его при
создании сервиса. Она покрывает запуск, управление аккаунтом, рабочие пространства на базе
организаций, API-ключи, локализацию, настройки среды выполнения, процесс качества и историю
релизов.

## Что входит в шаблон

- **Основа приложения**: Next.js App Router, React, TypeScript, Tailwind CSS v4, shadcn/ui,
  серверные действия, оболочку настроек и структуру функциональных срезов.
- **Аккаунты и авторизация**: Better Auth, показ только настроенных OAuth-провайдеров, настройки
  аккаунта, подключенные аккаунты, сессии, локальную авторизацию для автоматизации и защищенные
  маршруты.
- **Рабочие пространства**: рабочие пространства на базе организаций Better Auth, маршруты
  `/w/:organizationKey/...`, настройки, участники, роли, команды, приглашения, списки разрешенных
  доменов email и онбординг для пользователя без рабочего пространства.
- **API-доступ**: персональные API-ключи и API-ключи рабочего пространства, одноразовый показ
  секрета, права, срок действия, ограничения частоты и стартовые маршруты `/api/v1` только для
  чтения.
- **Локализация**: инфраструктура `next-intl`, сообщения на английском и русском языках,
  локализованные метаданные и локализованный контент документации с каноническими URL.
- **Процесс качества**: требования OpenSpec, Jest, Playwright E2E, общие вспомогательные функции
  автоматизации, обновления публичной документации, релизы и еженедельные заметки.

<Callout title="О локализации документации" variant="info">
  Страницы документации могут иметь варианты `.en.md` и `.ru.md`. Маршрут остается каноническим;
  система документации выбирает вариант под настроенную локаль UI и показывает языковой маркер
  только при запасном контенте.
</Callout>
