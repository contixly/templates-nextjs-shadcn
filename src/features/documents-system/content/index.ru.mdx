# Документация шаблона

Эта документация помогает понять, что уже есть в шаблоне Next.js приложения и куда смотреть при
добавлении собственного продукта. Шаблон включает публичные и защищенные маршруты, рабочие
пространства на базе Better Auth Organizations, команды, приглашения, API-ключи, локализацию и
готовые проверки качества.

## Что входит в шаблон

- **Основа приложения**: Next.js App Router, React, TypeScript, Tailwind CSS v4, shadcn/ui,
  серверные actions и feature-sliced структура.
- **Аккаунты и авторизация**: Better Auth, OAuth-провайдеры, защищенные маршруты, настройки
  профиля, подключенные аккаунты и управление сессиями.
- **Рабочие пространства**: workspace как пользовательский термин для Better Auth Organizations,
  маршруты `/w/:organizationKey/...`, настройки, участники, команды и приглашения.
- **API-доступ**: персональные и workspace API-ключи, одноразовый показ секрета, scopes,
  rate limits и стартовые read-only маршруты `/api/v1`.
- **Локализация**: инфраструктура `next-intl`, сообщения для `en` и `ru`, локализованные metadata
  и локализованные страницы документации.
- **Качество**: Jest, Playwright E2E, OpenSpec-backed сценарии и локальный automation auth для
  безопасной браузерной проверки.

<Callout title="О локализации документации" variant="info">
  Страницы документации могут иметь варианты `.en.md` и `.ru.md`. При открытии `/docs` система
  выбирает вариант под текущую локаль интерфейса, а если нужного варианта нет, показывает fallback с
  языковым маркером.
</Callout>

## Как читать этот раздел

Начните с [рабочего пространства](/docs/workspace), если оцениваете product flows, роли, команды и
приглашения. Используйте [глоссарий](/docs/general/glossary), когда встречаете термин из шаблона.
Историю изменений смотрите в [релизах](/docs/history/releases) и
[weekly notes](/docs/history/change-logs).
