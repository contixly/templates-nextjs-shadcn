# Управление API-ключами

API-ключи позволяют интеграциям вызывать `/api/v1` без браузерной сессии. Шаблон разделяет
персональные ключи и ключи рабочего пространства, чтобы пользователь осознанно выбирал субъект
доступа.

## Создать персональный ключ

1. Откройте `/user/api-keys`.
2. Нажмите **Создать ключ**.
3. Введите имя, по которому будет понятно назначение интеграции.
4. Выберите наборы прав, срок действия и ограничения частоты запросов.
5. Скопируйте секрет, когда он появится.

Секрет показывается один раз. Сохраните его в хранилище секретов интеграции до закрытия диалога.

## Создать ключ рабочего пространства

1. Откройте настройки рабочего пространства.
2. Перейдите в **API-ключи**.
3. Создайте ключ из раздела, принадлежащего рабочему пространству.

Ключи рабочего пространства действуют от имени одной организации. Они видны только тогда, когда
текущий участник имеет право читать API-ключи рабочего пространства или управлять ими.

## Изменить ключ

Редактирование ключа может менять имя, права, срок действия или ограничения частоты запросов.
Продление срока действия выполняется явно, чтобы устаревший ключ не продлевался незаметно.

## Удалить ключ

Удаляйте ключи, которые больше не используются. Персональные ключи отзываются при удалении
аккаунта владельца. Ключи рабочего пространства отзываются при удалении организации рабочего
пространства.
