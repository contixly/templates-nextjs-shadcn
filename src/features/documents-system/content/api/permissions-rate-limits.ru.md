# Права и ограничения частоты

API-ключи намеренно настраиваемые. Ключ должен получать только те права и срок жизни, которые нужны
его интеграции.

## Наборы прав

Наборы прав - это переиспользуемые группы областей доступа. Стартовый шаблон использует наборы для
API организаций в режиме только чтения. Продуктовая команда может добавить наборы для собственных
API-маршрутов.

Наборы раскрываются на сервере перед созданием или изменением ключа. Неизвестные наборы
отклоняются, а пользовательские права сохраняются при изменении несвязанных полей.

## Области доступа

Области доступа - это конкретные права, которые проверяют обработчики API-маршрутов. Маршрут
должен требовать самую точную область, описывающую возвращаемые или изменяемые данные.

Стартовая поверхность `/api/v1` работает только на чтение. Когда продукт добавляет маршруты
записи, добавьте явные области записи и задокументируйте их эффект до публикации.

## Срок действия

API-ключи могут истекать через заданный срок или создаваться без срока действия. Продление срока
выполняется явно при редактировании.

Используйте короткий срок для временной автоматизации и более длинный срок только для стабильных
интеграций между серверами с процессом ротации.

## Ограничения частоты

У каждого ключа могут быть максимальное число запросов и временное окно. Запросы после превышения
лимита получают `429` в стабильном формате API-ошибки.
