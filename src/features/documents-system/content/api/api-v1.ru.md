# Справочник API v1

Стартовые маршруты `/api/v1` публичны с точки зрения браузерной сессии, но требуют валидный
API-ключ. Передавайте ключ в заголовке `x-api-key`.

```bash
curl -H "x-api-key: $API_KEY" http://localhost:3000/api/v1/me
```

## Формат ответа

Успешные ответы используют поле `data`:

```json
{
  "data": {}
}
```

Обработанные ошибки используют поле `error`:

```json
{
  "error": {
    "code": "unauthorized",
    "message": "API key is required."
  }
}
```

## Маршруты

| Метод и путь | Назначение |
| --------------- | ---------- |
| `GET /api/v1/me` | Возвращает определенный субъект доступа API-ключа. |
| `GET /api/v1/organizations` | Возвращает организации, видимые субъекту доступа. |
| `GET /api/v1/organizations/:organizationId` | Возвращает одну видимую организацию. |
| `GET /api/v1/organizations/:organizationId/members` | Возвращает участников одной организации. |
| `GET /api/v1/organizations/:organizationId/teams` | Возвращает команды одной организации. |
| `GET /api/v1/organizations/:organizationId/teams/:teamId/members` | Возвращает участников одной команды. |

## Правила субъекта доступа

Персональные ключи определяются как пользовательский субъект доступа. Они могут читать данные
организации только пока пользователь остается участником этой организации.

Ключи рабочего пространства определяются как субъект доступа организации. Они читают только
организацию, которой принадлежит ключ.

## Частые ошибки

| Status | Значение |
| ------ | -------- |
| `401` | Ключ отсутствует или невалиден. |
| `403` | Ключ валиден, но не имеет права на маршрут или организацию. |
| `404` | Запрошенная организация или команда не видна ключу. |
| `429` | Ключ превысил ограничение частоты запросов. |
