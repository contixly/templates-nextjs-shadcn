# Локальная автоматизация и E2E

Шаблон включает локальный сценарий авторизации Better Auth для Playwright и браузерной
автоматизации. Он создает временных пользователей с активной сессией без небезопасных упрощений для
боевого окружения.

## Включить локальную автоматизацию

Задавайте `LOCAL_AUTOMATION_AUTH_ENABLED=true` только при локальной разработке. Для тестов, которым
нужно свежее чтение сессий, также задавайте `AUTH_DISABLE_SESSION_COOKIE_CACHE=true`.

Не включайте локальную авторизацию для автоматизации в боевом окружении.

## Создать пользователя для автоматизации

E2E-тесты должны предпочитать `signInLocalAutomationUser(page)` из `e2e/support/local-auth`. Эта
вспомогательная функция создает пользователя для локальной автоматизации, входит через тот же
контекст браузера и возвращает данные сценария для теста.

Пользователи автоматизации используют адреса в пространстве имен email
`local-agent+...@local-agent.test`.

## Очистить данные

Используйте `cleanupLocalAutomationUser(page)` из того же аутентифицированного контекста браузера.
Очистка отказывается работать с обычными пользователями и удаляет локальные организации без
участников, созданные сценарием.

## Запустить E2E

`npm run e2e` по умолчанию сам запускает сервер разработки на настроенном базовом URL Playwright.
Используйте `PLAYWRIGHT_START_SERVER=false` только когда намеренно тестируете уже запущенное
приложение с теми же настройками локальной автоматизации.
