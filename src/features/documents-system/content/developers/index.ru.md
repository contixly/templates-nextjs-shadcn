# Для разработчиков

Этот раздел предназначен для команд, которые строят сервис на базе шаблона. Он описывает проектные
договоренности, важные при добавлении продуктовых функций, API-маршрутов, экранов настроек или
документации.

Публичная пользовательская документация остается основной поверхностью. Страницы для разработчиков
объясняют, как расширять шаблон и не ломать поведение, на которое опираются пользователи.

## Модель разработки

Шаблон следует архитектуре функциональных срезов. Маршруты приложения остаются тонкими в `src/app`, а
бизнес-логика, схемы, репозитории, действия и UI функций живут в `src/features`.

## Что держать согласованным

Когда меняется видимое поведение, обновляйте связанные поверхности вместе:

- раздел требований OpenSpec в `openspec/specs`;
- реализацию в `src/features` и тонкие маршруты в `src/app`;
- E2E-сценарии в `e2e/specs`;
- публичную документацию в `src/features/documents-system/content`;
- релизные или еженедельные заметки, когда изменение публикуется.
