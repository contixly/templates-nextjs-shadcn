# Template documentation

This documentation explains what the Next.js application template already provides and where to
look when you add your own product. The template includes public and protected routes,
Better Auth organization-backed workspaces, teams, invitations, API keys, localization, and
quality checks ready for extension.

## What the template includes

- **Application foundation**: Next.js App Router, React, TypeScript, Tailwind CSS v4, shadcn/ui,
  server actions, and a feature-sliced structure.
- **Accounts and auth**: Better Auth, OAuth providers, protected routes, profile settings,
  connected accounts, and session management.
- **Workspaces**: the product-facing workspace term backed by Better Auth Organizations,
  `/w/:organizationKey/...` routes, settings, members, teams, and invitations.
- **API access**: personal and workspace API keys, one-time secret reveal, scopes, rate limits,
  and starter read-only `/api/v1` routes.
- **Localization**: `next-intl` infrastructure, `en` and `ru` messages, localized metadata,
  and localized documentation pages.
- **Quality**: Jest, Playwright E2E, OpenSpec-backed scenarios, and local automation auth for
  reliable browser checks.

<Callout title="Documentation localization" variant="info">
  Documentation pages can have `.en.md` and `.ru.md` variants. When `/docs` is opened, the system
  chooses the variant that matches the current UI locale. If that variant is missing, it shows a
  fallback page with a language marker.
</Callout>
